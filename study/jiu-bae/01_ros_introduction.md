# ROS2로 시작하는 로봇 프로그래밍

## 1장. ROS 소개

### 1. ROS란 무엇인가?

**ROS (Robot Operating System)** 는 이름과 달리 운영체제가 아니라, 로봇 애플리케이션 개발을 위한 **오픈소스 미들웨어 플랫폼**입니다.  
ROS는 로봇 개발에 필요한 공통 기능을 미리 제공하여, 개발자가 **핵심 로직 구현에만 집중**할 수 있게 도와줍니다.

**ROS의 정의**

- **Robot Operating System**
- 로봇을 위한 **오픈소스 메타 운영 시스템**
- 기존 OS 위에서 동작하는 **미들웨어 계층**

**ROS가 제공하는 주요 기능**

- **하드웨어 추상화**
  - 다양한 센서·모터·액추에이터를 공통 인터페이스로 추상화
- **하위 디바이스 제어**
  - 모터 제어, 센서 데이터 수집 등 저수준 제어를 위한 인터페이스 제공
- **로봇 기능 구현을 위한 라이브러리**
  - 센싱, 인식, 지도 작성, 모션 플래닝 등 알고리즘 패키지
- **프로세스 간 메시지 패싱**
  - 노드 간 통신을 위한 토픽(Topic), 서비스(Service), 액션(Action) 인터페이스
- **패키지 관리**
  - 기능별 패키지 단위로 재사용·배포 가능
- **개발 및 디버깅 도구**
  - `rviz`, `rqt`, `Gazebo` 등 시각화·시뮬레이션 도구 제공

요약하면, **ROS는 로봇 애플리케이션 개발을 위한 표준화된 플랫폼**입니다.

---

### 2. ROS 아키텍처 개념

ROS 시스템은 **노드(Node)** 를 기본 단위로 하는 **분산 시스템 구조**로 동작합니다.

![ROS 노드-토픽 구조](https://docs.ros.org/en/jazzy/_images/Nodes-TopicandService.gif)

**ROS 시스템 구성 요소**

- **Node**
  - ROS에서의 최소 실행 단위(독립 프로세스)
  - 예: 센서 노드, 제어 노드, 인식 노드 등
- **Topic / Service / Action**
  - 노드 간 메시지 통신 인터페이스
  - Topic: 비동기 스트림 데이터
  - Service: 요청–응답 구조
  - Action: 장시간 수행 작업(취소·피드백 지원)
- **Parameter**
  - 노드 설정값을 저장하는 글로벌 파라미터 서버
- **Package**
  - 실행 파일, 라이브러리, 설정 파일 등을 포함하는 **배포 단위**

**ROS 아키텍처의 핵심 특징**

- **노드 기반 분산 시스템**
  - 각 기능이 독립적으로 실행되어, 장애 격리와 확장이 용이
- **메시지 기반 통신**
  - 명확한 인터페이스를 통해 데이터 흐름을 표준화
- **패키지 기반 구조**
  - 코드 재사용성과 배포 효율을 극대화

### 3. ROS Graph 개념

![ROS Graph 예시](https://docs.ros.org/en/rolling/_images/unhide.png)

ROS 시스템에서 모든 노드와 통신 관계는 **ROS Graph** 로 표현됩니다.

**ROS Graph가 표현하는 것**

- 노드(Node)
- 토픽(Topic)
- 서비스(Service)
- 액션(Action)

예를 들어,

- `camera_node` → `image_topic` → `perception_node`

와 같은 데이터 흐름이 그래프 형태로 표시됩니다.  
이를 통해 **로봇 시스템 전체의 데이터 흐름과 의존 관계를 직관적으로 이해**할 수 있습니다.

### 4. 메타 운영체제 (Meta Operating System)

![ROS 메타 운영체제 개념](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdna%2FuokYb%2FbtsKeDeXE6z%2FAAAAAAAAAAAAAAAAAAAAANRtaCIcd-2isJekXl1vHOdL8H85fs5f_gGI-hYV9iQp%2Fimg.png)

ROS는 커널을 가진 전통적인 OS가 아니라, **애플리케이션 간 상호작용을 돕는 미들웨어 레이어**입니다.

**기존 OS 위에서 동작**

- Linux(Ubuntu), Windows, macOS 등 기존 운영체제 위에 설치
- 프로세스 관리·파일 시스템·드라이버 등은 기존 OS가 담당

**ROS가 담당하는 역할**

- **하드웨어 추상화**
  - 복잡한 센서·액추에이터 통신을 추상화하여 개발자가 알고리즘에 집중
- **분산 시스템 관리**
  - 여러 컴퓨터와 장치를 하나의 로봇 시스템처럼 연결
- **통신 인프라 제공**
  - 노드 간 메시지 라우팅, 네임스페이스 관리 등

## 2. ROS의 목적과 핵심 철학

### 1) 핵심 철학

ROS의 핵심 철학은 다음 한 문장으로 요약됩니다.

> **"Don't reinvent the wheel" (바퀴를 다시 발명하지 마라)**  

이미 존재하는 우수한 기능과 패키지를 재사용하여,  
**문제의 본질(로봇 서비스·제품의 가치)에 집중**하는 것을 목표로 합니다.

ROS의 궁극적인 목표는 다음과 같습니다.

- **전 세계 개발자가 함께 협업할 수 있는 로보틱스 소프트웨어 생태계 구축**
- 연구·교육·산업이 공유 가능한 **공통 기반 플랫폼** 제공

### 2) ROS가 해결하려는 문제

- 로봇마다 **매번 통신·시각화·센서 드라이버를 처음부터 구현**해야 하는 비효율
- 연구실·기업마다 코드가 **폐쇄적으로 분리**되어 재사용이 어려운 상황
- 하드웨어와 소프트웨어가 복잡하게 얽혀 있어 **유지보수와 확장**이 어려운 구조

ROS는 이 문제를 다음과 같은 방식으로 해결합니다.

#### 시장 출시 시간(Time to Market) 단축

- 이미 검증된 통신, 시각화, 시뮬레이션 프레임워크를 활용
- 알고리즘과 서비스 로직 개발에 집중하여 **제품 출시 속도 향상**

#### 코드 재사용성과 패키지 관리

- 기능별로 분리된 수천 개의 패키지를 **필요한 만큼 조립**하여 사용
- 오픈소스 패키지를 기반으로 **자신만의 패키지를 확장** 가능

#### 상업 친화적 라이선스 (Apache 2.0)

- ROS2는 **Apache 2.0 라이선스**를 채택
- 지적 재산권 충돌 없이 상용 제품(Product)에 활용 가능
- 기업에서도 **안심하고 제품 라인에 도입**할 수 있는 법적 기반 제공

## 3. ROS 패키지 생태계

ROS는 **수천 개의 검증된 오픈소스 패키지**를 제공하며,  
이 패키지들을 **블록처럼 조립하여 로봇 시스템을 구성**할 수 있습니다.

### 대표 패키지 예시

- **Navigation2**
  - 2D/3D 환경에서의 경로 계획 및 자율 주행
- **MoveIt**
  - 로봇 팔 매니퓰레이션 및 모션 플래닝
- **Cartographer**
  - 환경 지도 작성 및 위치 추정(SLAM)
- **Gazebo**
  - 물리 엔진 기반 3D 로봇 시뮬레이터
- **RViz**
  - 센서 데이터 및 로봇 상태를 3D로 시각화하는 도구

이러한 패키지들을 조합하면, 실제 로봇이 없더라도 **시뮬레이션 기반 개발–테스트–디버깅**이 가능합니다.

## 4. ROS 개발 역사

ROS 프로젝트는 **2007년 Stanford AI Lab**에서 시작되었습니다.  
이후 오픈소스 프로젝트로 성장하며 현재는 **ROS2 중심 생태계**로 전환되었습니다.

### ROS 연혁

- **2007년**: ROS 프로젝트 시작 (Stanford AI Lab)
- **2010년**: ROS1 첫 공개
- **2014년**: ROS2 개발 시작
- **2017년**: ROS2 첫 공식 릴리즈
- **2020년**: ROS1 마지막 버전 릴리즈
- **현재**: ROS2를 중심으로 생태계 전환 진행 중

### 릴리즈 네이밍과 주기

- 각 릴리즈는 **알파벳 순서에 맞춘 코드네임** 사용
- 로고는 전통적으로 **거북이 심볼**을 활용
- 일반적으로 **1년에 한 번, 5월경 릴리즈**

### 지원(서포트) 정책

- 릴리즈마다 지원 기간은 다르지만,
  - 일반적으로 **릴리즈 후 최소 2년간 지원**
  - LTS(Long Term Support) 버전의 경우 더 긴 지원 기간 제공

ROS는 **연구용 플랫폼에서 출발하여, 산업용 로봇 플랫폼으로 확장**되었으며,  
현재는 자율주행, 물류, 서비스 로봇 등 다양한 분야에서 표준 플랫폼으로 사용되고 있습니다.
