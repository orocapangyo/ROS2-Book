# 5장 ROS 2의 중요 콘셉트

## 5.1 개요

ROS 2는 로봇 소프트웨어를 구성하는 여러 가지 개념들을 제공한다.  
이 장에서는 ROS 2를 이해하고 사용하기 위해 반드시 알아야 할 핵심 콘셉트들을 하나씩 살펴본다.

---

## 5.2 노드 (Node)

노드는 ROS 2의 가장 기본적인 실행 단위이다.  
각 노드는 독립적인 프로세스로 동작하며, 하나의 기능(예: 센서 읽기, 모터 제어, 좌표 변환 등)을 담당한다.

### 노드의 특징

- **단일 책임 원칙**: 하나의 노드는 하나의 역할에 집중한다.
- **독립성**: 각 노드는 독립적으로 실행되고, 크래시가 발생해도 다른 노드에 바로 영향을 주지 않는다.
- **재사용 가능성**: 동일한 노드를 여러 프로젝트에서 재사용할 수 있다.
- **다언어 지원**: Python, C++ 등 다양한 언어로 작성할 수 있다.

### 노드 구성 예시

```
/camera_node       → 카메라 이미지 퍼블리시
/lidar_node        → 라이다 데이터 퍼블리시
/slam_node         → 지도 생성 및 위치 추정
/navigation_node   → 경로 계획 및 이동 명령 발행
/motor_node        → 모터 제어
```

---

## 5.3 토픽 (Topic)

토픽은 노드 간 데이터를 비동기적으로 주고받는 통신 채널이다.  
한 노드가 메시지를 **퍼블리시(Publish)** 하면, 해당 토픽을 구독하는 다른 노드들이 **서브스크라이브(Subscribe)** 하여 데이터를 수신한다.

### 토픽의 특징

- **단방향 스트리밍**: 퍼블리셔 → 서브스크라이버 방향의 단방향 데이터 흐름
- **1:N 구조 지원**: 하나의 퍼블리셔가 여러 서브스크라이버에게 동시에 메시지 전달
- **비동기 통신**: 퍼블리셔와 서브스크라이버는 동시에 실행될 필요가 없다
- **메시지 타입**: 각 토픽은 정해진 메시지 타입을 사용한다 (예: `sensor_msgs/Image`, `geometry_msgs/Twist`)

### 활용 예시

| 토픽 이름           | 내용             | 주요 사용 노드       |
|---------------------|------------------|----------------------|
| `/camera/image_raw` | 카메라 원본 이미지 | camera_node → vision_node |
| `/scan`             | 라이다 스캔 데이터 | lidar_node → slam_node |
| `/cmd_vel`          | 이동 명령         | nav_node → motor_node |
| `/odom`             | 오도메트리 정보   | motor_node → slam_node |

## 5.4 서비스 (Service)

서비스는 노드 간 **요청(Request)과 응답(Response)** 방식의 동기 통신이다.  
특정 작업을 수행하고 그 결과를 돌려받아야 하는 경우에 사용한다.

### 서비스의 특징

- **동기 통신**: 클라이언트가 요청을 보내면 서버의 응답을 기다린다.
- **1:1 통신 구조**: 하나의 서버에 하나의 클라이언트가 요청하는 구조
- **단발성 호출**: 연속 데이터 스트리밍이 아닌, 필요할 때 한 번 호출하는 방식
- **서비스 타입**: `.srv` 파일로 요청/응답 구조를 정의한다

### 활용 예시

```
클라이언트: "맵을 저장해줘" → 서버: "맵 저장 완료" 응답
클라이언트: "카메라 리셋" → 서버: "리셋 성공 여부" 응답
클라이언트: "현재 배터리 상태 요청" → 서버: "배터리 잔량 값" 응답
```

## 5.5 액션 (Action)

액션은 서비스보다 더 복잡한 작업을 처리하기 위한 통신 방식이다.  
목표를 설정하고, 중간 진행 상황을 피드백으로 받으며, 최종 결과를 반환하는 구조이다.

### 액션의 구성 요소

| 구성 요소  | 설명                                  |
|------------|---------------------------------------|
| Goal       | 클라이언트가 서버에 전달하는 목표 값   |
| Feedback   | 실행 중 중간 경과를 클라이언트에 전달  |
| Result     | 작업 완료 후 최종 결과 반환           |

### 액션의 특징

- **비동기 장기 작업**: 시간이 걸리는 작업(예: 경로 이동)에 적합
- **중간 취소 가능**: 목표 실행 중에 취소 요청을 보낼 수 있다
- **상태 관리**: 작업의 상태를 추적할 수 있다

### 활용 예시

```
목표(Goal): "좌표 (5, 3)으로 이동"
피드백(Feedback): "현재 위치 (2.5, 1.5), 진행률 50%"
결과(Result): "목표 지점 도착 성공"
```

## 5.6 메시지 (Message)

메시지는 노드 간 데이터를 주고받기 위한 데이터 구조이다.  
ROS 2는 `.msg` 파일을 통해 메시지 타입을 정의하며, 다양한 표준 메시지를 제공한다.

### 주요 메시지 패키지

| 패키지                 | 설명                          |
|------------------------|-------------------------------|
| `std_msgs`             | 기본 데이터 타입 (int, string 등) |
| `sensor_msgs`          | 센서 데이터 (카메라, 라이다 등)  |
| `geometry_msgs`        | 좌표, 속도, 포즈 관련          |
| `nav_msgs`             | 맵, 오도메트리, 경로 관련      |
| `visualization_msgs`   | RViz 시각화용 마커 등         |

## 5.7 파라미터 (Parameter)

파라미터는 노드 실행 중에 설정값을 동적으로 관리할 수 있는 기능이다.  
코드를 수정하거나 재컴파일 없이 동작 설정을 변경할 수 있어 유지보수에 유리하다.

### 파라미터의 특징

- **동적 변경**: 노드 실행 중에 파라미터 값을 변경할 수 있다
- **타입 지원**: int, float, bool, string, 배열 등 다양한 타입 지원
- **명령어로 조작**: `ros2 param get`, `ros2 param set` 등으로 조작 가능

### 활용 예시

```bash
# 파라미터 조회
ros2 param get /camera_node publish_rate

# 파라미터 설정
ros2 param set /camera_node publish_rate 30
```

## 5.8 런치 파일 (Launch File)

런치 파일은 여러 노드를 동시에 실행하고 설정을 관리하는 파일이다.  
ROS 2에서는 Python 기반 런치 파일을 사용한다.

### 런치 파일의 역할

- 여러 노드를 한 번에 실행
- 각 노드의 파라미터를 일괄 설정
- 네임스페이스 지정
- 노드 의존성 및 실행 순서 관리

### 예시 구조

```python
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_robot',
            executable='camera_node',
            name='camera',
        ),
        Node(
            package='my_robot',
            executable='motor_node',
            name='motor',
        ),
    ])
```

## 5.9 패키지 (Package)

패키지는 ROS 2에서 코드를 구성하는 기본 단위이다.  
하나의 패키지에는 노드, 메시지 정의, 런치 파일, 설정 파일 등이 포함된다.

### 패키지의 구성 요소

| 파일/폴더          | 설명                             |
|--------------------|----------------------------------|
| `package.xml`      | 패키지 메타데이터 및 의존성 정의  |
| `CMakeLists.txt`   | C++ 빌드 설정 (ament_cmake 사용)  |
| `setup.py`         | Python 빌드 설정 (ament_python)   |
| `src/`             | C++ 소스 코드                    |
| `my_package/`      | Python 소스 코드                 |
| `launch/`          | 런치 파일                        |
| `config/`          | YAML 파라미터 파일               |

## 5.10 워크스페이스 (Workspace)

워크스페이스는 ROS 2 프로젝트의 작업 공간이다.  
여러 패키지를 하나의 디렉터리 아래에서 관리하고 빌드한다.

### 워크스페이스 구조

```
ros2_ws/
├── src/                ← 소스 코드 디렉터리
│   ├── my_package/
│   └── another_package/
├── build/              ← 빌드 결과물
├── install/            ← 설치된 파일
└── log/                ← 빌드 로그
```

### 빌드 및 환경 설정

```bash
# 빌드
colcon build

# 환경 설정 적용
source install/setup.bash
```

## 5.11 정리

ROS 2의 핵심 콘셉트를 요약하면 다음과 같다.

| 콘셉트     | 역할                              |
|------------|-----------------------------------|
| 노드       | 독립적인 실행 단위                |
| 토픽       | 비동기 메시지 스트리밍            |
| 서비스     | 요청-응답 동기 통신               |
| 액션       | 장기 작업의 목표-피드백-결과 구조 |
| 메시지     | 노드 간 데이터 구조 정의          |
| 파라미터   | 동적 설정값 관리                  |
| 런치 파일  | 멀티 노드 일괄 실행 설정          |
| 패키지     | 코드 구성 단위                    |
| 워크스페이스 | 전체 프로젝트 작업 공간         |

이러한 콘셉트들이 유기적으로 연결되어 ROS 2 기반의 로봇 시스템이 구성된다.
