# ROS 2 패키지 설계 (Python)

ROS 2 패키지 설계의 핵심은 무엇인가?  
토픽, 서비스, 액션, 파라미터 등 다양한 기능을 통합한 4개의 노드로 구성된 패키지를 빌드하고 실행하는 방법을 배우는 것이 중요하다.

---

## 1. 패키지 구성 요소 및 역할

ROS 2 패키지 설계의 핵심은 토픽, 서비스, 액션, 파라미터와 같은 다양한 ROS 2 기능을 파이썬으로 구현하고 통합하는 것이다.

이번 예제에서는 총 **4개의 노드**로 구성된 패키지를 다룬다.

| 노드 | 주요 기능 |
|------|-----------|
| `argument_node` | 변수 a, b를 토픽으로 퍼블리싱 |
| `operator_node` | 덧셈·뺄셈·곱셈·나눗셈 연산을 서비스로 요청 |
| `calculator_node` | 메인 함수 포함, 연산 수행 |
| `checker_node` | 액션 클라이언트로서 총 결과값 확인 및 임계값 초과 시 종료 |

패키지 구성 파일:
- `package.xml` — 의존성 선언
- `setup.py` — 파이썬 패키지 설정
- 파이썬 스크립트 파일들 (노드별)

예제 코드는 GitHub에 공개되어 있으며, 다운로드 후 빌드하고 실행하는 과정을 진행한다.

---

## 2. 예제 패키지 상세 설명

### 노드별 역할
- **`argument_node`**: 랜덤한 두 변수(a, b)를 토픽으로 퍼블리싱한다.
- **`operator_node`**: `argument_node`로부터 받은 토픽을 사용하여 사칙연산을 서비스로 요청하고 결과를 반환한다.
- **`checker_node`**: 액션 클라이언트로서 액션 서버와 통신하며, 특정 임계값(예: 50 또는 100)을 초과하면 액션을 종료한다. 연산이 진행될 때마다 현재 누적 결과를 피드백으로 제공한다.
- **`calculator_node`**: 메인 함수를 포함하며, 연산의 중심 역할을 담당한다.

### GitHub 저장소 구조
- 인터페이스 예제 코드와 파이썬/C++ 예제 코드가 모두 포함되어 있다.
- 파이썬 예제: `topic_service_action_python/` 폴더
- C++ 예제: 별도 폴더
- 패키지 내부 구조: `setup.py`, `package.xml`, `src/` 폴더 (노드별 파이썬 파일 포함)

> 향후 런치 파일, 파라미터 설정(YAML 파일) 등에 대한 설명도 추가될 예정이다.

## 3. 패키지 빌드 및 설정 파일 이해

### package.xml
- 다른 패키지(예: 인터페이스 패키지)를 사용하려면 해당 패키지가 **미리 빌드**되어 있어야 한다.
- 의존성(dependency) 설정이 빌드의 핵심이다.

### setup.py 주요 섹션

| 섹션 | 역할 |
|------|------|
| `data_files` | 빌드·설치 시 `install` 폴더에 포함될 파일 지정 (런치 파일, 파라미터 파일 등) |
| `entry_points` | 각 노드의 진입점(entry point) 정의 (파일 경로 및 `main` 함수 명시) |

- `share/<package_name>/` 경로에 `.launch.py`, `.launch.xml`, `.yaml` 파일 등이 저장된다.
- `glob` 함수를 사용하여 특정 패턴(예: `*.launch.py`, `*.yaml`)에 맞는 파일들을 리스트업할 수 있다.

### 주요 빌드 명령어
```bash
# 의존성 설치
rosdep install --from-paths src --ignore-src -r -y

# 전체 빌드
colcon build

# 특정 패키지만 빌드
colcon build --packages-select <package_name>
```

빌드 후 파이썬 스크립트는 `install/lib/python3.x/site-packages/`에 저장된다.

## 4. 노드 실행 및 런치 파일 활용

### 노드 실행
```bash
ros2 run <package_name> <executable_name>
# 예시
ros2 run calculator_node calculator_node
```

### 노드 실행 흐름
1. `argument_node`가 랜덤 숫자를 생성하여 토픽으로 발행한다.
2. `calculator_node`가 토픽을 받아 연산을 수행한다.
3. `operator_node`가 서비스 요청으로 연산자와 함께 계산 결과를 반환한다.
4. `checker_node`가 액션 서버와 통신하며, 총합이 임계값(예: 50)을 초과할 때까지 연산을 요청하고 매 연산마다 피드백을 받는다.

### 런치 파일
- 여러 노드를 한 번에 실행하고, 파라미터 설정·조건부 실행 등 복잡한 구성을 가능하게 한다.
- 파라미터는 YAML 파일로 관리할 수 있다.

```bash
ros2 launch <package_name> <launch_file_name>
```

> 향후 런치 파일, 파라미터, 실행 인자 등을 다루는 방법에 대한 추가 설명이 제공될 예정이다.
