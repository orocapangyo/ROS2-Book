# ROS 2 토픽 프로그래밍 (Python)

ROS 2에서 토픽 프로그래밍은 어떻게 파이썬으로 구현할 수 있을까?  
ROS 2 토픽은 **비동기식 단방향 통신** 방식으로, 퍼블리셔(발행자)와 서브스크라이버(구독자) 노드를 생성하여 메시지를 주고받는 것이 핵심이다.

---

## 1. 토픽 통신 기본 개념

| 개념 | 설명 |
|------|------|
| 토픽(Topic) | ROS 2에서 메시지를 주고받는 통신 방식 |
| 비동기식(Asynchronous) | 메시지를 보내는 시점과 받는 시점이 일치하지 않아도 됨 |
| 단방향(One-way) | 메시지는 한쪽 방향으로만 흐름 |
| 퍼블리셔(Publisher) | 토픽으로 메시지를 발행(송신)하는 노드 |
| 서브스크라이버(Subscriber) | 토픽으로 메시지를 수신하는 노드 |

- 기본적으로 **일대일** 통신 구조이지만, **일대다(One-to-many)** 형태로도 구성 가능하다.
- 토픽 통신은 ROS 프로그램에서 가장 많이 사용되는 통신 방식 중 하나이다.

---

## 2. 퍼블리셔 노드 구현 (Python)

### 메시지 인터페이스 정의
- `ros_tutorials_topic` 패키지 내 `msg/` 폴더에 `ArithmeticMessage.msg` 파일로 메시지 형식을 정의한다.
- 메시지 구성: `timestamp` (시간 정보), `a`, `b` (두 개의 정수 데이터)
- 컴파일 과정을 거쳐 파이썬 코드에서 읽어들일 수 있는 형태로 변환된다.

### 노드 초기화 및 파라미터 설정
- `Node` 클래스를 상속받아 노드를 생성한다.
- `__init__` 함수에서 노드 이름을 설정하고, `declare_parameter`로 ROS 파라미터 서버에 파라미터를 등록한다.
  - 예시 파라미터: `ros_namespace`, `random_number_min`, `random_number_max`
- `update_parameter` 멤버 함수: 파라미터 서버에서 파라미터가 변경될 때 호출되며, 데이터 타입과 값의 유효성을 검증한다.

### 퍼블리셔 생성 및 메시지 발행

**QoS 설정 예시**
```python
# depth=10, reliability='reliable', durability='transient_local'
# history='keep_last', lifespan=10초
```

**퍼블리셔 생성**
```python
self.publisher = self.create_publisher(ArithmeticMessage, 'message_arithmetic', qos)
```

**타이머 설정 (1초마다 발행)**
```python
self.timer = self.create_timer(1.0, self.publish_message)
```

**`publish_message` 함수 동작**
1. `ArithmeticMessage` 타입의 메시지 객체를 생성한다.
2. 메시지에 현재 시간(`timestamp`)과 0~9 사이의 랜덤 정수 `a`, `b` 값을 할당한다.
3. `publisher.publish(message)`를 통해 메시지를 발행한다.
4. `self.get_logger().info()`로 발행된 메시지 정보를 로그로 출력한다.

## 3. 서브스크라이버 노드 구현 (Python)

### 토픽 구독 설정
```python
self.subscription = self.create_subscription(
    ArithmeticMessage,
    'message_arithmetic',
    self.get_arithmetic_message,
    qos,
    callback_group=self.enter_callback_group
)
```

### 콜백 그룹과 QoS의 역할

| 구분 | 설명 |
|------|------|
| 콜백 함수 | 토픽 메시지를 받아서 처리하는 함수 |
| `SingleThreadedExecutor` (기본) | 하나의 콜백 함수만 실행, 현재 콜백 종료 후 다음 실행 |
| `MultiThreadedCallbackGroup` | 여러 콜백 함수를 동시에 병렬로 실행 |
| `MultiThreadedExecutor` | 병렬 처리를 위해 `MultiThreadedCallbackGroup`과 함께 사용 |

- `MultiThreadedCallbackGroup`은 `MultiThreadedExecutor`와 함께 사용될 때 병렬 실행이 가능하다.

### 콜백 함수 구현 (`get_arithmetic_message`)
- 수신한 `ArithmeticMessage`에서 `timestamp`, `a`, `b` 데이터를 추출한다.
- `a`와 `b` 값을 멤버 변수에 저장한다.
- `get_logger().info()`로 수신한 메시지 내용을 로그로 출력한다.

## 4. 노드 실행 및 엔트리 포인트

### setup.py 엔트리 포인트
- **`argument` 노드**: `argument` 패키지의 `list/` 폴더 안 `argument.py`의 `main` 함수 실행
- **`calculator` 노드**: `calculator` 패키지 내 `main.py`의 `main` 함수 실행 (`calculator.py`는 `main.py`에서 사용)

### argument 노드 실행 흐름
```
rclpy 초기화
  └─ Argument 클래스 인스턴스 생성
      └─ rclpy.spin() 으로 노드 유지
          └─ KeyboardInterrupt 발생 시
              └─ 종료 메시지 출력 → rclpy.shutdown()
```

### calculator 노드 실행 흐름 (멀티 스레드)
```python
executor = MultiThreadedExecutor()   # 스레드 수 미지정 시 CPU 수에 맞춰 자동 설정
executor.add_node(node)
executor.spin()
# KeyboardInterrupt 발생 시
executor.shutdown()
rclpy.shutdown()
```

- `calculator` 노드에는 **액션 서버**도 포함되어 있어, 종료 시 액션 서버 정리 함수도 실행된다.
- 여러 노드를 모아놓은 것을 **풀(Pool)**이라 하며, 스레드 하나당 하나의 노드를 실행한다.

---

## 5. 요약

| 역할 | 구현 순서 |
|------|-----------|
| 퍼블리셔 | 노드 설정 → QoS 정의 → `create_publisher` → 메시지 발행 함수 작성 |
| 서브스크라이버 | 노드 설정 → QoS 정의 → `create_subscription` → 콜백 함수 작성 |

- `argument` 노드(퍼블리셔)와 `calculator` 노드(서브스크라이버) 간 토픽 통신을 통해 랜덤 데이터를 주고받는 구조를 구현하였다.
- 멀티 스레드 환경에서는 `MultiThreadedExecutor`와 `MultiThreadedCallbackGroup`을 함께 사용하여 콜백 함수들을 병렬로 처리할 수 있다.
