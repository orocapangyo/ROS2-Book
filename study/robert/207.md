# ROS 2 서비스 프로그래밍 (Python)

ROS 2에서 Python으로 서비스 프로그래밍을 하려면 어떻게 해야 할까?  
서비스는 **요청-응답(Request-Response)** 방식으로, 서버와 클라이언트 코드를 각각 작성해야 하며, 클라이언트 코드에서는 비동기 통신을 위해 `spin_once`와 `wait_for_service`를 활용하는 것이 핵심이다.

---

## 1. 서비스 통신 개요

| 구분 | 설명 |
|------|------|
| 통신 방식 | 요청-응답(Request-Response) |
| 서비스 클라이언트 | 서비스를 요청하는 역할 |
| 서비스 서버 | 요청을 받아 처리하고 응답하는 역할 |

### 서비스 vs 액션

| 구분 | 서비스 | 액션 |
|------|--------|------|
| 구조 | 요청 → 응답 (단순) | 요청 → 피드백 → 최종 결과 |
| 중간 피드백 | 없음 | 있음 |
| 취소 기능 | 없음 | 있음 |

> 파라미터 서버, 파라미터 메시지 통신도 서비스의 확장된 개념으로 볼 수 있다.

## 2. 예제 코드 구조

| 노드 | 역할 |
|------|------|
| `operator_node` | 서비스 클라이언트 — 연산자 정보를 요청 |
| `calculate_operator_node` | 서비스 서버 — 연산 수행 후 결과 반환 |

## 3. 서비스 메시지 인터페이스 정의

파일명: `ROS2_message_operator.srv`

```
# 요청(Request)
int32 a
int32 b
int32 operator   # 1=+, 2=-, 3=*, 4=/

---
# 응답(Response)
float32 result
```

- 서비스 메시지 인터페이스 내에 `ADD`, `SUBTRACT`, `MULTIPLY`, `DIVIDE`와 같은 **상수**가 정의되어 사용된다.

## 4. 서비스 서버 코드 분석

### 서비스 생성
```python
self.service = self.create_service(
    ROS2MessageOperator,       # 서비스 메시지 타입
    'ROS2_message_operator',   # 서비스 ID
    self.get_service_operator, # 콜백 함수
    callback_group=self.callback_group
)
```

### 콜백 함수 (`get_service_operator`) 동작
1. `request.operator` 값을 멤버 변수에 저장한다.
2. `operator` 값(1~4)에 따라 덧셈·뺄셈·곱셈·나눗셈 멤버 함수를 호출한다.
3. 나눗셈 시 **0으로 나누는 경우** 예외 처리를 수행한다.
4. 정의되지 않은 연산자 값에 대한 `else` 처리도 포함한다.
5. 계산된 결과를 `response.result`에 할당하여 반환하고, 연산 과정을 로그로 출력한다.

### 서버 메인 코드
- `calculate_operator` 노드는 토픽, 서비스, 액션 등 여러 작업을 처리하므로 **`MultiThreadedExecutor`** 를 사용한다.

| Executor | 특징 |
|----------|------|
| `SingleThreadedExecutor` | 하나의 작업만 순차 처리 |
| `MultiThreadedExecutor` | 여러 작업을 동시에 처리, 복잡한 노드에 적합 |

## 5. 서비스 클라이언트 코드 분석

### 클라이언트 생성
```python
self.client = self.create_client(
    ROS2MessageOperator,
    'ROS2_message_operator'
)
```

### 서비스 가용성 확인
```python
while not self.client.wait_for_service(timeout_sec=0.2):
    self.get_logger().error('서비스가 준비되지 않았습니다.')
```

### 서비스 요청 (비동기)
```python
request = ROS2MessageOperator.Request()
request.operator = random_operator   # 랜덤 연산자 설정

self.future = self.client.call_async(request)
```

### 동기 vs 비동기 호출

| 방식 | 함수 | 설명 |
|------|------|------|
| 동기 | `call_sync` | 응답이 올 때까지 블로킹 |
| 비동기 | `call_async` | 요청 후 다른 작업 수행, 응답이 오면 처리 |

- `spin_once`는 ROS 2 관련 작업을 한 번 처리하며, 비동기 통신에서 `wait_for_service`와 함께 활용된다.

### 응답 처리
```python
if self.future.done():
    try:
        response = self.future.result()
    except Exception as e:
        self.get_logger().error(f'서비스 호출 실패: {e}')
    else:
        self.get_logger().info(f'결과: {response.result}')
        self.user_trigger = False
```

### 메인 함수 실행 흐름
```
rclpy 초기화
  └─ operator 노드 생성
      └─ ros_ok() 루프
          ├─ user_trigger=True → 서비스 요청 (call_async)
          ├─ spin_once() → ROS 2 작업 처리
          └─ 응답 수신 완료 → user_trigger=False, 엔터 대기
  └─ KeyboardInterrupt → 노드 종료 → rclpy.shutdown()
```

## 6. 예외 처리 및 코드 검사

### `# NOQA` 주석
- "No Quality Assurance"의 약자로, 특정 라인에 대한 코드 검사(linting)를 건너뛰도록 지시한다.
- 예: `pylint` 등이 사용하지 않는 변수에 대해 경고를 표시하는 것을 방지한다.

### `try-except-else-finally` 구문

| 블록 | 실행 조건 |
|------|-----------|
| `try` | 예외가 발생할 수 있는 코드 실행 |
| `except` | `try` 블록에서 예외 발생 시 실행 |
| `else` | `try` 블록에서 예외가 없을 때 실행 |
| `finally` | 예외 발생 여부와 상관없이 항상 실행 |
