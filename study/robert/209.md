# ROS 2 파라미터 프로그래밍 (Python)

ROS 2에서 파라미터 프로그래밍은 어떻게 활용될까?  
노드 실행 시 **YAML 파일**을 통해 파라미터를 초기화하거나, **CLI** 및 **서비스 클라이언트**를 통해 동적으로 변경하며 로봇 시스템의 유연성을 높일 수 있다.

---

## 1. 파라미터의 개념 및 활용

### 파라미터 주요 기능

| 방법 | 설명 |
|------|------|
| 코드 내 변경 | 프로그램 내에서 직접 파라미터 값을 변경 |
| CLI (`ros2 param`) | 커맨드라인 인터페이스로 파라미터 조회·변경 |
| 서비스 클라이언트 | 외부 노드에서 서비스 통신으로 set/get |
| YAML 파일 | 노드 실행 시 파라미터 일괄 초기화 |

### 서비스와 파라미터의 차이

| 구분 | 서비스 | 파라미터 |
|------|--------|----------|
| 메시지 인터페이스 | 사용자 정의 | ROS 2에서 이미 정의된 인터페이스 사용 |
| 통신 방식 | 요청-응답 | 파라미터 서버-클라이언트 (서비스 기반) |

---

## 2. 파라미터 관련 Python 코드 구성 요소

| 함수 | 역할 |
|------|------|
| `declare_parameter` | 노드에서 사용할 파라미터를 미리 선언 |
| `get_parameter` | 파라미터 값을 읽어옴 (값이 없으면 기본값 사용) |
| `add_on_set_parameter_callback` | 외부에서 파라미터 변경 시 실행될 콜백 함수 등록 |

> 세 함수 모두 노드 클래스 내 멤버 함수이다.

---

## 3. argument 노드 예시 코드 분석

### 노드 초기화 및 파라미터 선언

```python
# 파라미터 선언 (기본값 10)
self.declare_parameter('qos_depths', 10)

# 파라미터 값 가져오기
qos_depths = self.get_parameter('qos_depths').value

# 파라미터 변경 콜백 등록
self.add_on_set_parameter_callback(self.update_parameters)
```

- YAML 파일이나 CLI로 설정된 값이 있으면 해당 값이 사용되고, 없으면 기본값(10)이 사용된다.

### 노드에서 사용하는 주요 파라미터

- `qos_depths`
- `min_random_number`
- `max_random_number`

### `update_parameters` 콜백 함수 동작

```
parameters 리스트 수신 (name + type 정보 포함)
  └─ 각 파라미터 순회
      ├─ min_random_number → 멤버 변수 업데이트
      ├─ max_random_number → 멤버 변수 업데이트
      └─ 유효성 검증 (설정값이 허용 범위 초과 시 변경 거부 가능)
  └─ SetParametersResult(successful=True) 반환
```

- `min_random_number`, `max_random_number`는 랜덤 숫자 생성 범위(a, b)를 설정하며, 파라미터 변경을 통해 생성 범위를 조정할 수 있다.

## 4. CLI를 이용한 파라미터 관리

### 노드 실행
```bash
ros2 run topic_service_examples arguments
```
- 기본 파라미터: `min_random_number=0`, `max_random_number=9` (0~9 사이 랜덤 생성)

### `ros2 param` 명령어

| 명령어 | 설명 |
|--------|------|
| `ros2 param list` | 실행 중인 노드의 파라미터 목록 조회 |
| `ros2 param get <노드> <파라미터>` | 특정 파라미터의 타입과 현재 값 확인 |
| `ros2 param set <노드> <파라미터> <값>` | 파라미터 값 변경 |

```bash
# 예시: max_random_number를 100으로 변경
ros2 param set /arguments max_random_number 100
# 출력: Set parameter successful
```

- 변경 성공 시 `set_parameter successful` 메시지가 출력된다.
- 이후 0~100 사이의 랜덤 값이 생성된다.

## 5. 서비스 클라이언트를 이용한 파라미터 관리

각 ROS 2 노드는 자체적인 **파라미터 서버**를 가지며, 외부 노드에서 서비스 클라이언트를 통해 파라미터를 set/get 할 수 있다.

### 서비스 클라이언트 생성

```python
from rcl_interfaces.srv import SetParameters
from rcl_interfaces.msg import Parameter, ParameterValue, ParameterType

self.param_client = self.create_client(
    SetParameters,
    '/arguments/set_parameters'
)
```

### 파라미터 설정 요청

```python
request = SetParameters.Request()

param_value = ParameterValue(
    type=ParameterType.PARAMETER_INTEGER,
    integer_value=100
)
request.parameters = [
    Parameter(name='max_random_number', value=param_value)
]

# 비동기 호출
future = self.param_client.call_async(request)
```

- `SetParameters` 인터페이스는 ROS 2에서 이미 정의된 파라미터 서비스 인터페이스이다.
- 비동기 방식을 주로 사용하는 이유: 설정 과정의 시간 지연을 줄이기 위함이다.

## 6. YAML 파일을 이용한 파라미터 초기화

### YAML 파일 구조 예시

```yaml
/**:
  ros__parameters:
    qos_depths: 30
    min_random_number: 0
    max_random_number: 9
```

- `/**`: 현재 설정된 네임스페이스나 노드 이름을 자동으로 적용한다.
  - 노드 이름이나 네임스페이스가 변경되어도 자동으로 매핑되어 유연하게 적용 가능하다.
- 노드 실행 시 이 파일을 읽어 파라미터를 초기화한다.
- CLI로 변경한 파라미터는 **메모리 상의 변경**이므로, 노드 재시작 시 YAML 파일의 초기값으로 복원된다.

### 파라미터 파일 위치

- `examples/checker/parameter/rs_medic_config.yml`

## 7. 런치 파일을 통한 파라미터 설정

```python
# rs_medic.launch.py 예시
param_file = LaunchConfiguration('param_file',
    default=os.path.join(get_package_share_directory('package_name'),
                         'parameter', 'rs_medic_config.yml'))

Node(
    package='package_name',
    executable='arguments',
    parameters=[param_file]
)
```

- `launch_configuration`으로 YAML 파일 경로를 지정한다.
- 노드 실행 시 해당 YAML 파일을 읽어 파라미터를 초기화한다.

## 8. 패키지 빌드 및 설치 시 파일 복사

빌드·설치 시 YAML 파일도 함께 복사되어야 런치 파일에서 정상적으로 참조할 수 있다.

```python
# setup.py 예시
data_files=[
    ('share/' + package_name + '/launch', glob('launch/*.launch.py')),
    ('share/' + package_name + '/parameter', glob('parameter/*.yaml')),
],
```

- `data_files`에 런치 폴더와 파라미터 폴더의 파일을 `share/<패키지 이름>/`으로 복사하도록 지정한다.
