# ROS 2 실행 인자 프로그래밍 (Python)

ROS 2에서 파이썬으로 실행 인자를 어떻게 프로그래밍할까?  
`argparse` 모듈을 활용하면 프로그램 수정 없이 노드 실행 시 파라미터 값을 동적으로 변경하여 유연하게 로봇을 제어할 수 있다.

---

## 1. 실행 인자(Argument)의 개념과 필요성

### 실행 인자란?
프로그램이나 노드를 실행할 때 추가적인 값을 전달하여, **프로그램 코드를 수정하지 않고도** 내부의 특정 값(예: 파라미터)을 변경할 수 있게 하는 기능이다.

### 실행 인자의 필요성
- 예: ROS 2의 체커 노드를 실행할 때, 액션 클라이언트가 특정 값(예: 50)까지 더하면 멈추도록 설정하는데, 이 값을 프로그램 수정 없이 동적으로 변경하고 싶을 때 사용한다.
- `ros2 run` 명령어 뒤에 `-g`와 같은 옵션을 사용하여 값을 전달함으로써, 노드 실행 조건을 다르게 설정할 수 있다.

### 용어 정리
| 용어 | 별칭 |
|------|------|
| 실행 인자 | 아규먼트(Argument) |
| 파라미터 | 매개변수 |

---

## 2. 파이썬에서의 실행 인자 처리 방법: argparse

### C++와 파이썬의 차이점
| 언어 | 처리 방식 |
|------|-----------|
| C++ | `argc`(인자 개수), `argv`(인자 값 배열) 사용 |
| Python | `argparse` 모듈 사용 |

### 파이썬 실행 인자 처리 과정
1. `rclpy`를 초기화할 때 `sys.argv`를 통해 전달된 인자들을 `argparse`로 넘겨준다.
2. `sys.argv`의 첫 번째 요소(인덱스 0)는 실행 파일의 경로와 이름이므로, **실제 인자 값은 인덱스 1부터** 시작한다.
3. `argparse`는 전달받은 문자열 형태의 인자들을 파싱하여 사용하기 쉬운 형태로 변환한다.

### argparse 모듈 활용 예시
```python
import argparse
import sys

# 파서 생성
parser = argparse.ArgumentParser()

# 실행 인자 추가 (-g / --total_sum)
parser.add_argument('-g', '--total_sum', type=int, default=9, help='목표 합계 값')

# 인자 파싱
args = parser.parse_args(sys.argv[1:])

# 파싱된 값 사용
print(args.total_sum)
```

---

## 3. argparse를 이용한 실행 인자 추가 및 사용

### 실행 인자 추가 방법
- `add_argument()` 메서드를 사용하여 실행 인자를 정의한다.
- **짧은 이름**: 하이픈 하나(`-`) 사용 → 예: `-g`
- **긴 이름**: 하이픈 두 개(`--`) 사용 → 예: `--total_sum`

### 파싱 및 값 사용
- `parser.parse_args()` 호출 → 정의된 실행 인자들이 파싱되어 `args` 객체에 저장된다.
- `args.total_sum`을 통해 `-g` 또는 `--total_sum`으로 전달된 값에 접근한다.

### 실행 인자의 기본값 및 활용
- 실행 인자를 지정하지 않았을 경우를 대비하여 **기본값(default)**을 설정할 수 있다.
  - 예: `default=9` → 인자 미입력 시 9 사용
- 실행 시 인자를 전달하면(예: `-g 100`) 기본값 대신 해당 값으로 노드 동작이 변경된다.

---

## 4. argparse 사용 흐름 요약

```
1. 파서 생성
   └─ argparse.ArgumentParser()

2. 인자 추가
   └─ add_argument(짧은이름, 긴이름, type, default, help)

3. 인자 파싱
   └─ args = parser.parse_args(sys.argv[1:])

4. 인자 분리
   └─ 노드에서 사용할 인자(예: -g)와
      rclpy.init()에 전달할 나머지 인자를 분리

5. 인자 사용
   └─ args.total_sum 등으로 값에 접근
```
