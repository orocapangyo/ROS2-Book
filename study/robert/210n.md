## 슬라이드 2. 실행 인자의 개념과 필요성

먼저 실행 인자가 무엇인지 보겠습니다.

실행 인자는 프로그램이나 노드를 실행할 때 추가로 전달하는 값입니다.

이처럼 실행 인자는 같은 프로그램을 여러 조건에서 유연하게 실행할 수 있게 해주는 중요한 기능입니다.

## 슬라이드 3. 실행 인자가 필요한 상황

실행 인자가 필요한 이유를 조금 더 구체적으로 보겠습니다.

로봇 프로그램은 테스트 환경과 실제 운용 환경에서 설정값이 달라질 수 있습니다.  
예를 들어 시뮬레이션에서는 작은 목표값으로 빠르게 테스트하고, 실제 실행에서는 더 큰 목표값을 사용할 수 있습니다.

이때 매번 코드를 수정하면 실수 가능성이 커지고, 관리도 어려워집니다.

실행 인자를 사용하면 동일한 코드를 유지하면서 실행 조건만 바꿀 수 있습니다.

예를 들어 다음과 같이 실행할 수 있습니다.

`ros2 run <package_name> <executable_name> -g 50`

또는 목표값을 바꿔서 다음처럼 실행할 수도 있습니다.

`ros2 run <package_name> <executable_name> -g 100`

즉, 실행 인자는 프로그램의 재사용성과 테스트 편의성을 높여주는 도구입니다.

## 슬라이드 5. Python에서 실행 인자 처리 방법

실행 인자를 처리하는 방식은 언어마다 다릅니다.

C++에서는 보통 `argc`와 `argv`를 사용합니다.  
`argc`는 전달된 인자의 개수이고, `argv`는 실제 인자 값들이 들어 있는 배열입니다.

Python에서도 `sys.argv`를 통해 실행 인자에 접근할 수 있습니다.  
하지만 `sys.argv`만 직접 사용하면 문자열 파싱, 타입 변환, 기본값 처리 등을 모두 직접 구현해야 합니다.

그래서 Python에서는 일반적으로 `argparse` 모듈을 사용합니다.

`argparse`를 사용하면 실행 인자의 짧은 이름, 긴 이름, 타입, 기본값, 도움말을 체계적으로 정의할 수 있습니다.

ROS 2 Python 노드에서도 실행 인자를 다룰 때 `argparse`를 사용하면 코드가 훨씬 명확해지고 유지보수하기 쉬워집니다.

## 슬라이드 6. sys.argv와 인자 처리 구조

Python 프로그램을 실행하면 실행 명령어에 포함된 값들이 `sys.argv`에 저장됩니다.

여기서 중요한 점은 `sys.argv`의 첫 번째 값, 즉 인덱스 0번에는 실행 파일의 이름이나 경로가 들어간다는 것입니다.

실제로 사용자가 입력한 인자 값은 인덱스 1번부터 시작합니다.

예를 들어 다음과 같이 실행했다고 가정해보겠습니다.

`python main.py -g 100`

이 경우 `sys.argv[0]`에는 `main.py`가 들어갑니다.  
`sys.argv[1]`에는 `-g`, `sys.argv[2]`에는 `100`이 들어갑니다.

따라서 argparse로 실제 실행 인자만 처리하려면 보통 `sys.argv[1:]`을 넘겨줍니다.

이 구조를 이해해야 ROS 2 노드에서 사용자 정의 인자와 ROS 2 내부 인자를 적절히 구분할 수 있습니다.

## 슬라이드 7. argparse 기본 예제

이제 `argparse`를 사용하는 기본 예제를 보겠습니다.

먼저 `argparse`와 `sys` 모듈을 import합니다.

그다음 `argparse.ArgumentParser()`를 사용해서 파서 객체를 생성합니다.

이 파서 객체에 `add_argument()`를 사용해 실행 인자를 추가합니다.

예제에서는 `-g`와 `--total_sum`이라는 두 가지 이름을 사용합니다.  
`-g`는 짧은 옵션이고, `--total_sum`은 긴 옵션입니다.

또한 `type=int`를 지정해 입력값을 정수로 변환하고, `default=9`를 지정해 인자가 없을 때 기본값 9를 사용하도록 합니다.

마지막으로 `parser.parse_args(sys.argv[1:])`를 호출하면 실행 인자가 파싱되어 `args` 객체에 저장됩니다.

그 후 `args.total_sum`으로 전달된 값을 사용할 수 있습니다.

## 슬라이드 8. add_argument 사용 방법

`add_argument()`는 실행 인자를 정의하는 핵심 메서드입니다.

먼저 짧은 이름을 지정할 수 있습니다.  
짧은 이름은 하이픈 하나를 사용합니다.  
예를 들어 `-g`처럼 작성합니다.

다음으로 긴 이름을 지정할 수 있습니다.  
긴 이름은 하이픈 두 개를 사용합니다.  
예를 들어 `--total_sum`처럼 작성합니다.

짧은 이름은 빠르게 입력하기 좋고, 긴 이름은 의미를 명확하게 보여준다는 장점이 있습니다.

또한 `type` 옵션을 통해 입력값의 자료형을 지정할 수 있습니다.  
실행 인자는 기본적으로 문자열로 들어오기 때문에 숫자로 쓰려면 `type=int`와 같이 변환을 지정해야 합니다.

`default`는 사용자가 값을 입력하지 않았을 때 사용할 기본값입니다.  
`help`는 해당 옵션의 설명입니다.

이 옵션들을 잘 사용하면 명령어만 봐도 어떤 값을 전달하는지 쉽게 이해할 수 있습니다.

## 슬라이드 9. parse_args와 값 사용

실행 인자를 정의한 뒤에는 `parse_args()`를 호출해 실제 입력값을 분석합니다.

예제에서는 `args = parser.parse_args(sys.argv[1:])` 형태로 사용합니다.

이렇게 하면 사용자가 입력한 `-g 100` 또는 `--total_sum 100` 같은 값이 분석되어 `args` 객체에 저장됩니다.

그다음에는 `args.total_sum`처럼 속성 형태로 값에 접근합니다.

예를 들어 사용자가 `-g 100`을 입력했다면 `args.total_sum` 값은 100이 됩니다.

반대로 사용자가 아무 인자도 전달하지 않았다면 `default=9`에 의해 값은 9가 됩니다.

이렇게 얻은 값은 ROS 2 노드 안에서 목표 합계값, 반복 횟수, 임계값 같은 실행 조건으로 사용할 수 있습니다.

즉, `parse_args()`는 명령어로 입력된 문자열을 프로그램에서 사용할 수 있는 값으로 바꿔주는 단계입니다.

## 슬라이드 10. 기본값과 실행 시 값 변경

실행 인자에서 기본값은 매우 중요합니다.

항상 사용자가 모든 옵션을 입력한다고 가정하면 프로그램이 불편해집니다.  
그래서 일반적인 실행 조건은 기본값으로 정해두고, 필요할 때만 실행 인자로 값을 바꾸는 방식이 좋습니다.

예제에서는 `default=9`가 설정되어 있습니다.

따라서 사용자가 아무 값을 입력하지 않으면 목표 합계값은 9로 동작합니다.

하지만 실행 시 `-g 100`을 전달하면 기본값 9 대신 100이 사용됩니다.

즉, 기본값은 기본 실행 조건을 제공하고, 실행 인자는 그 값을 필요에 따라 덮어쓰는 역할을 합니다.

이 구조를 사용하면 같은 노드를 수정 없이 다양한 조건으로 실행할 수 있습니다.

## 슬라이드 11. argparse 사용 흐름 요약

지금까지의 흐름을 단계별로 정리해보겠습니다.

첫 번째 단계는 파서 생성입니다.  
`argparse.ArgumentParser()`를 사용해 실행 인자를 처리할 파서를 만듭니다.

두 번째 단계는 인자 추가입니다.  
`add_argument()`를 사용해 짧은 이름, 긴 이름, 타입, 기본값, 도움말을 정의합니다.

세 번째 단계는 인자 파싱입니다.  
`parse_args(sys.argv[1:])`를 호출해 실제 입력된 값을 분석합니다.

네 번째 단계는 인자 분리입니다.  
ROS 2에서는 노드에서 사용할 인자와 `rclpy.init()`에 전달해야 할 나머지 인자를 구분해야 합니다.

다섯 번째 단계는 인자 사용입니다.  
예를 들어 `args.total_sum` 값을 읽어 노드의 목표값으로 전달합니다.

이 흐름을 이해하면 ROS 2 Python 노드에 실행 인자를 쉽게 추가할 수 있습니다.
