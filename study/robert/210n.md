```md
## 슬라이드 2. 실행 인자의 개념

먼저 실행 인자가 무엇인지 보겠습니다.

실행 인자는 프로그램이나 노드를 실행할 때 추가로 전달하는 값입니다.  
즉, 프로그램 코드 안에 고정된 값을 직접 바꾸지 않고, 실행 명령어에서 값을 전달해 내부 동작을 바꾸는 방식입니다.

예를 들어 어떤 노드가 목표 합계값을 기준으로 동작한다고 가정해보겠습니다.  
기본값은 50이지만, 어떤 테스트에서는 100까지 계산하고 싶을 수 있습니다.

이때 코드 안의 값을 50에서 100으로 직접 수정하지 않고, 실행할 때 `-g 100`과 같은 옵션을 전달하면 됩니다.

이렇게 하면 같은 코드로 여러 실행 조건을 만들 수 있습니다.  
즉, 실행 인자는 프로그램의 재사용성과 테스트 편의성을 높여주는 기능입니다.

## 슬라이드 3. 실행 인자가 필요한 이유

실행 인자가 필요한 이유는 명확합니다.

첫째, 코드를 수정하지 않고 실행 조건을 바꿀 수 있습니다.  
로봇 시스템에서는 테스트 환경, 시뮬레이션 환경, 실제 운용 환경마다 설정값이 달라질 수 있습니다.

둘째, 실험을 반복하기 쉽습니다.  
예를 들어 목표값을 10, 50, 100으로 바꿔가며 동작을 비교할 수 있습니다.

셋째, 노드의 유연성이 높아집니다.  
하나의 노드를 여러 상황에서 재사용할 수 있기 때문입니다.

이번 예제에서는 ROS 2의 체커 노드에서 목표 합계값을 실행 인자로 전달하는 상황을 다룹니다.  
즉, 액션 클라이언트가 특정 값까지 더하면 멈추도록 할 때, 그 기준값을 실행 시점에 바꾸는 것입니다.

## 슬라이드 4. 실행 인자와 파라미터 용어 정리

여기서 용어를 간단히 정리하겠습니다.

이번 강의에서는 특히 실행 명령어에서 값을 전달하는 실행 인자 처리에 집중하겠습니다.

## 슬라이드 5. C++와 Python의 실행 인자 처리 차이

실행 인자를 처리하는 방식은 언어마다 조금 다릅니다.

C++에서는 보통 `argc`와 `argv`를 사용합니다.  
`argc`는 전달된 인자의 개수를 의미하고, `argv`는 실제 인자 값들이 담긴 배열입니다.

반면 Python에서는 일반적으로 `argparse` 모듈을 사용합니다.

물론 Python에서도 `sys.argv`를 통해 실행 인자에 접근할 수 있습니다.  
하지만 `sys.argv`를 직접 처리하면 문자열을 직접 해석하고, 타입 변환과 기본값 처리도 직접 해야 합니다.

`argparse`를 사용하면 이런 작업을 훨씬 편하게 처리할 수 있습니다.  
짧은 옵션 이름, 긴 옵션 이름, 타입, 기본값, 도움말까지 한 번에 정의할 수 있습니다.

따라서 ROS 2 Python 노드에서는 실행 인자 처리를 위해 `argparse`를 사용하는 것이 일반적입니다.

## 슬라이드 6. sys.argv와 인자 구조

Python 프로그램을 실행하면 실행 명령어에 포함된 값들이 `sys.argv`에 저장됩니다.

여기서 중요한 점은 `sys.argv`의 첫 번째 요소, 즉 인덱스 0번에는 실행 파일의 경로나 이름이 들어간다는 것입니다.

실제로 사용자가 입력한 실행 인자 값은 인덱스 1번부터 시작합니다.

예를 들어 프로그램을 다음과 같이 실행했다고 가정해보겠습니다.

`python main.py -g 100`

이 경우 `sys.argv[0]`에는 `main.py`가 들어가고, `sys.argv[1]`에는 `-g`, `sys.argv[2]`에는 `100`이 들어갑니다.

그래서 argparse로 실제 인자만 처리하려면 보통 `sys.argv[1:]`을 전달합니다.

이 구조를 이해해야 ROS 2 노드에서 사용자 인자와 ROS 2 자체 인자를 적절히 분리할 수 있습니다.

## 슬라이드 7. argparse 기본 사용법

이제 `argparse`의 기본 사용법을 보겠습니다.

먼저 `argparse.ArgumentParser()`를 사용해서 파서 객체를 생성합니다.

그다음 `add_argument()` 메서드로 사용할 실행 인자를 정의합니다.

예를 들어 목표 합계값을 받기 위해 `-g`와 `--total_sum` 옵션을 추가할 수 있습니다.

여기서 `-g`는 짧은 이름이고, `--total_sum`은 긴 이름입니다.

또한 `type=int`를 지정하면 입력받은 값을 정수로 변환합니다.  
`default=9`를 지정하면 사용자가 값을 입력하지 않았을 때 기본값 9를 사용합니다.

마지막으로 `parser.parse_args()`를 호출하면 입력된 인자들이 파싱되어 `args` 객체에 저장됩니다.

---

## 슬라이드 8. add_argument의 주요 옵션

`add_argument()`에서 자주 사용하는 옵션을 조금 더 자세히 보겠습니다.

첫 번째는 짧은 이름입니다.  
하이픈 하나를 사용하며, 예를 들어 `-g`처럼 작성합니다.  
명령어를 짧게 입력할 수 있다는 장점이 있습니다.

두 번째는 긴 이름입니다.  
하이픈 두 개를 사용하며, 예를 들어 `--total_sum`처럼 작성합니다.  
옵션의 의미를 명확히 보여줄 수 있습니다.

세 번째는 `type`입니다.  
실행 인자는 기본적으로 문자열로 들어오기 때문에, 숫자로 사용하려면 `type=int`처럼 타입을 지정해야 합니다.

네 번째는 `default`입니다.  
사용자가 인자를 입력하지 않았을 때 사용할 기본값입니다.

다섯 번째는 `help`입니다.  
해당 인자가 어떤 역할을 하는지 설명하는 문구입니다.

이 옵션들을 잘 설정하면 실행 인자 사용성이 좋아집니다.

## 슬라이드 9. 실행 인자 파싱과 값 사용

실행 인자를 정의한 뒤에는 `parse_args()`를 호출해서 값을 파싱합니다.

예제에서는 `args = parser.parse_args(sys.argv[1:])` 형태로 사용합니다.

이렇게 하면 사용자가 입력한 실행 인자가 분석되어 `args` 객체에 저장됩니다.

이후에는 `args.total_sum`처럼 속성 형태로 값에 접근할 수 있습니다.

예를 들어 사용자가 `-g 100` 또는 `--total_sum 100`을 입력했다면, `args.total_sum` 값은 100이 됩니다.

반대로 사용자가 아무 값도 입력하지 않았다면, add_argument에서 지정한 기본값인 9가 사용됩니다.

이렇게 파싱된 값은 노드 내부에서 목표값이나 설정값으로 활용할 수 있습니다.

## 슬라이드 10. 기본값 default의 의미

실행 인자를 사용할 때 기본값은 매우 중요합니다.

기본값이 없으면 사용자가 인자를 입력하지 않았을 때 프로그램이 오류를 낼 수 있습니다.

하지만 `default=9`처럼 기본값을 지정해두면, 실행 인자가 없어도 프로그램이 정상적으로 동작합니다.

예를 들어 체커 노드에서 목표 합계값의 기본값을 9로 설정했다고 가정하겠습니다.

사용자가 별도의 인자를 주지 않으면 목표값 9를 사용합니다.  
하지만 `-g 100`을 전달하면 목표값은 100으로 바뀝니다.

즉, 기본값은 일반적인 실행 조건을 제공하고, 실행 인자는 필요할 때 그 값을 덮어쓰는 역할을 합니다.

이 구조를 사용하면 테스트와 실제 실행을 모두 유연하게 처리할 수 있습니다.

## 슬라이드 11. ROS 2에서 실행 인자와 rclpy 인자 분리

ROS 2 Python 노드에서는 실행 인자를 처리할 때 한 가지 더 주의해야 합니다.

사용자가 전달한 인자 중에는 노드에서 직접 사용할 인자도 있지만, ROS 2 내부에서 사용하는 인자도 있을 수 있습니다.

따라서 노드에서 사용할 인자와 `rclpy.init()`에 전달할 인자를 분리하는 과정이 필요합니다.

예를 들어 `-g`나 `--total_sum`은 우리가 직접 정의한 인자입니다.  
반면 ROS 2 remapping이나 namespace 관련 인자는 rclpy가 처리해야 할 수 있습니다.

이때 argparse로 사용자 정의 인자를 먼저 처리하고, 나머지 인자를 rclpy 쪽으로 넘기는 방식으로 구성할 수 있습니다.

이 분리 과정을 잘못 처리하면 ROS 2 실행 옵션과 사용자 정의 옵션이 충돌할 수 있습니다.

따라서 ROS 2 Python 실행 인자 처리에서는 argparse와 rclpy 초기화 흐름을 함께 고려해야 합니다.

## 슬라이드 12. argparse 사용 흐름 요약

argparse 사용 흐름을 전체적으로 정리해보겠습니다.

첫 번째 단계는 파서 생성입니다.  
`argparse.ArgumentParser()`로 파서 객체를 만듭니다.

두 번째 단계는 인자 추가입니다.  
`add_argument()`를 사용해 짧은 이름, 긴 이름, 타입, 기본값, 도움말을 정의합니다.

세 번째 단계는 인자 파싱입니다.  
`parse_args()`를 호출해서 사용자가 입력한 값을 분석합니다.

네 번째 단계는 인자 분리입니다.  
노드에서 사용할 인자와 `rclpy.init()`에 전달할 ROS 2 인자를 구분합니다.

다섯 번째 단계는 인자 사용입니다.  
예를 들어 `args.total_sum` 값을 읽어서 노드의 목표값으로 전달합니다.

이 흐름을 익혀두면 다양한 ROS 2 Python 노드에 실행 인자를 쉽게 추가할 수 있습니다.

이렇게 실행하면 노드 내부에서는 `args.total_sum` 값이 100으로 설정되고, 해당 값에 맞춰 동작이 달라집니다.

넷째, 코드의 재사용성이 높아집니다.  
하나의 노드를 여러 목적에 맞게 사용할 수 있기 때문입니다.

따라서 실행 인자는 단순한 옵션 처리 기능이 아니라, ROS 2 노드를 더 유연하게 만드는 설계 도구라고 볼 수 있습니다.

## 슬라이드 15. 마무리

이번 시간에는 ROS 2에서 Python으로 실행 인자를 처리하는 방법을 살펴보았습니다.

핵심은 `argparse` 모듈을 사용한다는 점입니다.

먼저 파서를 만들고, `add_argument()`로 실행 인자를 정의합니다.  
그다음 `parse_args()`로 입력값을 파싱하고, `args.total_sum`처럼 값에 접근합니다.

실행 인자는 코드 수정 없이 노드의 동작 조건을 바꿀 수 있게 해줍니다.  
예를 들어 체커 노드의 목표 합계값을 실행 시점에 `-g` 또는 `--total_sum`으로 전달할 수 있습니다.

또한 ROS 2에서는 사용자 정의 인자와 rclpy에서 처리할 인자를 구분하는 것이 중요합니다.

이 구조를 이해하면 Python 기반 ROS 2 노드를 훨씬 유연하게 설계하고 실행할 수 있습니다.

이상으로 ROS 2 실행 인자 프로그래밍 Python 발표를 마치겠습니다.  
감사합니다.
```
