# ROS 프로그래밍 규칙 (ROS Programming Rules)

## 제 1장: 가독성과 협업 효율을 높이는 코드 스타일

이 문서는 ROS(Robot Operating System) 프로그래밍 시 일관된 가독성과 유지보수성을 유지하기 위한 코드 스타일 가이드를 정리한 자료입니다.

---

### 핵심 목표 (Key Objectives)

* **코드 품질 유지**: 가독성을 높여 협업 효율을 극대화하고 코드 리뷰 시간을 단축합니다.  
* **유지보수 용이성**: 표준화된 규칙을 통해 시스템의 장기적인 안정성을 확보합니다.  
* **디자인 컨셉**: Friendly Learning (배경 `#E3F2FD`, 강조 `#FF7043`, 폰트 Quicksand/Nunito)

---

### 1.2. 기본 이름 규칙 (Naming Conventions)

기본적으로 모든 파일과 변수명은 명확성과 일관성을 우선합니다.

| 분류 | 규칙 | 예시 |
| :---- | :---- | :---- |
| **파일, 변수, 함수** | `snake_case` (소문자 \+ 언더바) | `motor_controller.cpp`, `max_speed` |
| **타입, 클래스** | `CamelCased` (대문자 시작) | `SensorData`, `PathPlanner` |
| **상수** | `UPPERCASE` (전체 대문자) | `MAX_SPEED = 10` |

#### 도구별 고유명사 (예외 사항)

일부 파일은 특정 빌드 시스템이나 도구의 요구에 따라 고유한 이름을 유지합니다.

* `package.xml`  
* `CMakeLists.txt`  
* `README.md`  
* `.gitignore`  
* `LICENSE`, `CHANGELOG.rst`

---

### 1.3. C++ 스타일 가이드 (C++ Style Guide)

ROS 2 개발은 기본적으로 **C++14 표준**을 준수합니다.

#### 주요 포맷팅 규칙

1. **들여쓰기(Indent)**: 탭(Tab) 대신 **2 Space**를 사용합니다.  
2. **라인 길이**: 가독성을 위해 한 줄은 최대 **100문자**를 넘지 않도록 합니다.  
3. **접근 지정자**: `public`, `protected`, `private`은 들여쓰기를 하지 않습니다.  
4. **괄호(Brace)**: `if`, `else`, `for` 등 모든 제어문에는 중괄호를 사용합니다.

#### C++ 이름 규칙 상세

* **CamelCased**: 타입, 클래스, 구조체, 열거형  
* **snake\_case**: 패키지, 네임스페이스, 변수, 함수, 메소드  
* **g\_ (접두사)**: 전역 변수  
* **\_ (접미사)**: 클래스 멤버 변수 (예: `sensor_value_`)  
* **확장자**: 소스 파일은 `.cpp`, 헤더 파일은 `.hpp`를 권장합니다.

#### 린터 및 분석 도구

* **스타일 검사**: `ament_cpplint`, `ament_uncrustify`  
* **정적 분석**: `ament_cppcheck`

---

### 1.4. 파이썬 스타일 가이드 (Python Style Guide)

파이썬 개발은 **Python 3.5 이상**을 사용하며, **PEP 8** 표준을 준수합니다.

* **들여쓰기**: **4 Space**를 사용하며, 긴 인수는 내어쓰기(Hanging indent)를 적용합니다.  
* **이름 규칙**:  
  * `CamelCased`: 타입, 클래스  
  * `snake_case`: 모듈, 함수, 메소드, 변수, 패키지  
* **린터**: `ament_flake8`을 사용하여 스타일 일관성을 검사합니다.

---

### 1.5. 요약 및 참고 사항

* **C언어**: C99 standard 및 REP-7 준수  
* **JavaScript**: Airbnb Javascript Style Guide 참고  
* **스터디 팁**: 발표 시 C++(2칸)과 Python(4칸)의 들여쓰기 차이점을 강조하고, 멤버 변수의 접미사(`_`) 규칙을 실제 코드 예시와 함께 설명하면 효과적입니다.
