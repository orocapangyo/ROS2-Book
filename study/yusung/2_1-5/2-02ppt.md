## \[슬라이드 2\] 2.1. ROS의 Hello World, rclpy 버전

### ROS 2 통신의 기초: 가장 단순한 토픽(Topic) 통신 구현

* **핵심 개념:** ROS 2 통신의 기본이 되는 \*\*Topic(토픽)\*\*을 파이썬(`rclpy`)으로 구현합니다.  
* **학습 목표:**  
  * Hello World 메시지를 지속적으로 보내는 노드(Publisher) 만들기  
  * 해당 메시지를 받아서 출력하는 노드(Subscriber) 만들기  
* **사용 언어 및 라이브러리:** Python 3, `rclpy` (ROS 2 Python Client Library)

---

## \[슬라이드 3\] 2.2. 패키지 생성

### ROS 2 프로젝트의 시작: 기초 뼈대 만들기

* **패키지 생성 명령어 (`ros2 pkg create`)**  
  * `--build-type ament_python`: 파이썬 빌드 시스템 지정  
  * `--dependencies rclpy std_msgs`: 사용할 의존성 패키지(파이썬 라이브러리, 표준 메시지) 지정

$ cd \~/robot\_ws/src/

$ ros2 pkg create my\_first\_ros\_rclpy\_pkg \--build-type ament\_python \--dependencies rclpy std\_msgs

* **생성 결과:** `setup.py`, `package.xml` 등 설정 파일과 소스코드 폴더가 자동 생성됩니다.

---

## \[슬라이드 4\] 2.3. 패키지 설정

### 실행 환경 구성: 의존성 및 실행 스크립트 설정

* **`package.xml`:** 패키지 정보 및 의존성(`rclpy`, `std_msgs`)이 정의된 문서. 빌드 타입이 `<build_type>ament_python</build_type>`으로 설정됩니다.  
* **`setup.py` (핵심 설정):**  
  * `ros2 run`으로 실행할 \*\*진입점(Entry Point)\*\*을 설정합니다.  
  * `console_scripts` 항목에 퍼블리셔와 서브스크라이버의 실행 명령어를 매핑합니다.

'console\_scripts': \[

    'helloworld\_publisher \= my\_first\_ros\_rclpy\_pkg.helloworld\_publisher:main',

    'helloworld\_subscriber \= my\_first\_ros\_rclpy\_pkg.helloworld\_subscriber:main',

\],

## \[슬라이드 5\] 2.4. 퍼블리셔 노드 작성

### 메시지 발신자: 퍼블리셔(Publisher) 클래스 구현

* **역할:** 특정 토픽(`helloworld`)으로 문자열 메시지를 지속적으로 발행.  
* **주요 구성 요소:**  
  * `Node` 클래스 상속  
  * `create_publisher(메시지타입, 토픽이름, QoS)`: 퍼블리셔 생성  
  * `create_timer(시간, 콜백함수)`: 일정한 주기(예: 1초)마다 메시지를 발행하도록 타이머 설정

\# 퍼블리셔 생성 예시

self.helloworld\_publisher \= self.create\_publisher(String, 'helloworld', qos\_profile)

---

## \[슬라이드 6\] 2.5. 서브스크라이버 노드 작성

### 메시지 수신자: 서브스크라이버(Subscriber) 클래스 구현

* **역할:** 토픽(`helloworld`)에 발행되는 메시지를 실시간으로 수신하여 로그로 출력.  
* **주요 구성 요소:**  
  * `create_subscription(메시지타입, 토픽이름, 콜백함수, QoS)`: 구독자 생성  
  * **콜백 함수(Callback):** 메시지가 도착할 때마다 자동으로 실행되는 처리 함수

\# 서브스크라이버 생성 예시

self.helloworld\_subscriber \= self.create\_subscription(

    String, 'helloworld', self.subscribe\_topic\_message, qos\_profile)

## \[슬라이드 7\] 2.6. 빌드 (Build)

### 패키지 컴파일: 작성한 코드를 실행 가능한 상태로 빌드

* **Colcon 빌드 시스템 사용:** 워크스페이스 루트(`~/robot_ws`)에서 빌드 진행  
* `--symlink-install`: 파이썬 스크립트 수정 시 재빌드 없이 바로 적용되도록 심볼릭 링크로 설치  
* `--packages-select`: 특정 패키지만 선택하여 빠르게 빌드

$ cd \~/robot\_ws

$ colcon build \--symlink-install \--packages-select my\_first\_ros\_rclpy\_pkg

* **환경 변수 적용:** 빌드 후 반드시 `. ~/robot_ws/install/local_setup.bash` 실행 필요

## \[슬라이드 8\] 2.7. 실행 (Run)

### 노드 구동: 퍼블리셔와 서브스크라이버 노드 실행 확인

* **실행 명령어 (`ros2 run`)**  
  * `ros2 run [패키지명] [실행노드명]` 형식을 사용합니다.

\# 터미널 1 (퍼블리셔 실행)

$ ros2 run my\_first\_ros\_rclpy\_pkg helloworld\_publisher

\[INFO\]: Published message: Hello World: 0

\# 터미널 2 (서브스크라이버 실행)

$ ros2 run my\_first\_ros\_rclpy\_pkg helloworld\_subscriber

\[INFO\]: Received message: Hello World: 0

* **결과:** 터미널 창 두 개를 열어, 한쪽에서 메시지를 발행하면 다른 쪽에서 즉시 수신하는 것을 확인할 수 있습니다.
