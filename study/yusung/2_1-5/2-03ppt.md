## \[슬라이드 2\] 3.1. ROS의 Hello World, rclcpp버전

### rclpy와의 차이점 및 rclcpp 개요

* **핵심 개념:** 파이썬(`rclpy`) 환경과 대비되는 C++ 환경(`rclcpp`)에서의 기본 통신(Topic) 구현 방식을 알아봅니다.  
* **학습 목표:**  
  * ROS 2 C++ 클라이언트 라이브러리인 `rclcpp`의 구조 파악  
  * C++의 객체 지향적 특성을 활용한 노드(Node) 설계 기본기 다지기

## \[슬라이드 3\] 3.2. 패키지 생성

### ROS 2 C++ 프로젝트의 기초 뼈대 만들기

* **패키지 생성 명령어 (`ros2 pkg create`)**  
  * `--build-type ament_cmake`: C++ 빌드 시스템(`ament_cmake`) 지정  
  * `--dependencies rclcpp std_msgs`: C++ 라이브러리 및 표준 메시지 의존성 추가

$ cd \~/robot\_ws/src/

$ ros2 pkg create my\_first\_ros\_rclcpp\_pkg \--build-type ament\_cmake \--dependencies rclcpp std\_msgs

* **생성 결과:** `include/`, `src/` 폴더와 함께 `CMakeLists.txt`, `package.xml` 설정 파일이 생성됩니다.

---

## \[슬라이드 4\] 3.3. 패키지 설정

### 실행 환경 구성: 빌드(ament\_cmake) 및 의존성 설정

* **`package.xml` 설정:**  
  * 패키지 메타정보 정의 및 `<buildtool_depend>ament_cmake</buildtool_depend>` 명시  
  * `rclcpp`, `std_msgs` 의존성 추가  
* **`CMakeLists.txt` (핵심 빌드 설정):**  
  * `find_package()`: 필요한 의존성 패키지 탐색  
  * `add_executable()`: C++ 소스 코드를 실행 파일(노드)로 빌드하도록 정의  
  * `ament_target_dependencies()`: 타겟 실행 파일과 의존성 패키지 연결  
  * `install(TARGETS ...)`: 빌드된 실행 파일의 설치 경로 지정

## \[슬라이드 5\] 3.4. 퍼블리셔 노드 작성

### 메시지 발신자: C++ 클래스를 활용한 퍼블리셔 구현

* **구현 방식:** `rclcpp::Node` 클래스를 상속받아 `HelloworldPublisher` 클래스 작성  
* **주요 구성 요소:**  
  * `create_publisher<std_msgs::msg::String>()`: 토픽명과 QoS를 지정하여 퍼블리셔 객체 생성  
  * `create_wall_timer()`: 일정한 주기(예: 1초, `1s`)마다 메시지를 발행하는 콜백 타이머 설정  
  * 스마트 포인터(`SharedPtr`)와 `std::bind` 또는 `lambda`를 활용한 메모리 및 콜백 관리

// 퍼블리셔 생성 예시

helloworld\_publisher\_ \= this-\>create\_publisher\<std\_msgs::msg::String\>("helloworld", qos\_profile);

---

## \[슬라이드 6\] 3.5. 서브스크라이버 노드 작성

### 메시지 수신자: 콜백 기반의 C++ 서브스크라이버 구현

* **구현 방식:** `rclcpp::Node`를 상속받은 `HelloworldSubscriber` 클래스 작성  
* **주요 구성 요소:**  
  * `create_subscription<std_msgs::msg::String>()`: 대상 토픽명, QoS, 콜백 함수를 바인딩하여 구독자 생성  
  * `std::bind` & `placeholders::_1`: 수신된 메시지 데이터를 콜백 함수의 인자로 안전하게 전달  
  * `RCLCPP_INFO()`: 로거를 사용하여 수신된 메시지를 화면에 출력

// 서브스크라이버 생성 예시

helloworld\_subscriber\_ \= this-\>create\_subscription\<std\_msgs::msg::String\>(

  "helloworld", qos\_profile, std::bind(\&HelloworldSubscriber::subscribe\_topic\_message, this, \_1));

## \[슬라이드 7\] 3.6. 빌드 (Build)

### 패키지 컴파일: Colcon을 이용한 C++ 패키지 빌드

* **Colcon 빌드 도구 사용:** 워크스페이스 최상단(`~/robot_ws`)에서 빌드 진행  
* **빌드 명령어 옵션:**  
  * `--packages-select`: 워크스페이스 내 특정 패키지만 지정하여 빠른 빌드 수행  
  * `--symlink-install`: 설치 디렉터리에 심볼릭 링크 생성 (빌드 속도 최적화)

$ cd \~/robot\_ws

$ colcon build \--symlink-install \--packages-select my\_first\_ros\_rclcpp\_pkg

* **환경 설정:** 빌드 완료 후 `. ~/robot_ws/install/local_setup.bash` 명령어로 워크스페이스를 반드시 갱신해야 합니다.

## \[슬라이드 8\] 3.7. 실행 (Run)

### 노드 구동: 컴파일된 C++ 노드의 통신 결과 확인

* **실행 명령어:** `ros2 run [패키지명] [실행파일_노드명]` 형식을 사용합니다.

\# 터미널 1 (서브스크라이버 대기)

$ ros2 run my\_first\_ros\_rclcpp\_pkg helloworld\_subscriber

\[INFO\]: Received message: 'Hello World: 0'

\# 터미널 2 (퍼블리셔 동작)

$ ros2 run my\_first\_ros\_rclcpp\_pkg helloworld\_publisher

\[INFO\]: Published message: 'Hello World: 0'

* **결과 확인:** 퍼블리셔가 발행하는 메시지를 서브스크라이버가 즉각적으로 수신하여 터미널에 로그로 출력하는 것을 확인할 수 있습니다.
