## \[슬라이드 2\] 4.1. 설정 스크립트(setup script)

### 워크스페이스 환경 적용의 기본

* **개요:** 빌드된 패키지를 ROS 2 시스템이 인식하고 실행할 수 있도록 환경 경로를 설정해 주는 스크립트입니다.  
* **적용 방법:** 터미널을 열 때마다 매번 실행할 필요 없이, `~/.bashrc` 파일 하단에 명령어 한 줄을 추가하여 자동으로 적용되게 하는 것이 효율적입니다.

\# 설치된 패키지 환경 적용 예시

$ source \~/robot\_ws/install/local\_setup.bash

## \[슬라이드 3\] 4.2. setup.bash vs local\_setup.bash

### 환경 초기화(Underlay)와 레이어 추가(Overlay)의 차이점

* **`setup.bash` (전체 환경 초기화 / Underlay):**  
  * 시스템에 설치된 ROS 2의 기본 환경 및 코어 패키지들을 로드합니다.  
  * 위치 예시: `/opt/ros/jazzy/setup.bash`  
* **`local_setup.bash` (현재 레이어만 추가 / Overlay):**  
  * 사용자가 직접 작업하고 빌드한 워크스페이스 내부의 패키지 환경만을 기존 환경 위에 덧붙입니다.  
* **💡 스터디 팁:** `~/.bashrc`에 시스템 `setup.bash`를 먼저 로드하고, 그 다음에 사용자의 `local_setup.bash`를 순차적으로 로드하도록 설정하면 패키지 충돌 없이 쾌적한 개발이 가능합니다.

## \[슬라이드 4\] 4.3. colcon\_cd

### 터미널 디렉토리 이동을 더 빠르고 간편하게

* **기능 소개:** 복잡한 경로를 모두 타이핑할 필요 없이, `colcon_cd` 명령어 뒤에 패키지 이름만 입력하면 해당 패키지의 소스 코드가 있는 디렉토리로 즉시 이동시켜 줍니다.  
* **설정 방법:** 터미널의 `~/.bashrc` 파일에 아래 내용을 추가하여 기능을 활성화합니다.

\# colcon\_cd 기능 활성화 및 기준이 되는 루트 워크스페이스 지정

$ source /usr/share/colcon\_cd/function/colcon\_cd.sh

$ export \_colcon\_cd\_root=\~/robot\_ws

\# 실제 사용 예시

$ colcon\_cd \[패키지이름\]

## \[슬라이드 5\] 4.4. ROS\_DOMAIN\_ID vs Namespace

### 동일 네트워크 상의 노드 충돌 방지 및 분리 기법

* **1\. 물리적 분리:** 독립적인 스위치 허브나 라우터를 사용하여 하드웨어적으로 네트워크 자체를 분리합니다.  
* **2\. `ROS_DOMAIN_ID` 변경:** DDS의 통신 도메인 그룹을 아예 분리하여 다른 그룹 간의 패킷 전송을 원천 차단합니다. (0\~101 사이의 숫자 사용)  
  * *예시:* `$ export ROS_DOMAIN_ID=11` (11번 도메인을 가진 기기끼리만 통신)  
* **3\. 네임스페이스(`__ns`) 추가:** 동일한 도메인 안에서 통신하되, 노드나 토픽 이름 앞에 논리적인 접두사를 붙여 대상을 구분합니다.

\# 네임스페이스를 다르게 주어 동일한 거북이 노드를 독립적으로 2개 실행

$ ros2 run turtlesim turtlesim\_node \--ros-args \-r \_\_ns:=/robot1

$ ros2 run turtlesim turtlesim\_node \--ros-args \-r \_\_ns:=/robot2

## \[슬라이드 6\] 4.5. 명령어 자동 완성 기능 추가

### colcon과 vcstool의 탭(Tab) 자동 완성 활성화

* **개요:** 터미널에서 `colcon` 빌드 도구나 `vcs` 버전 관리 툴을 사용할 때, 명령어 옵션이나 패키지 이름을 일일이 타이핑하지 않고 **Tab 키**를 눌러 빠르게 자동 완성할 수 있도록 돕는 유용한 환경 설정입니다.  
* **설정 방법:** `~/.bashrc` 파일 하단에 관련 스크립트를 추가하여 활성화합니다.

\# colcon 및 vcs 자동 완성 스크립트 적용

source /usr/share/colcon\_argcomplete/hook/colcon-argcomplete.bash

source /usr/share/vcstool-completion/vcs.bash
