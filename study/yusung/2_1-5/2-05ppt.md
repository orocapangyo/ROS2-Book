## \[슬라이드 2\] 5.1. ROS 2 인터페이스 신규 작성

### 통신 목적에 맞는 사용자 정의 데이터 타입 설계

* **개요:** ROS 2에서 제공하는 기본 메시지 타입 외에, 사용자가 직접 프로젝트 목적에 맞는 데이터 통신 규격을 정의할 수 있습니다.  
* **이번 장에서 생성할 3가지 커스텀 인터페이스:**  
  * **1\. 토픽 (Topic):** `ArithmeticArgument.msg` (지속적인 데이터 전송)  
  * **2\. 서비스 (Service):** `ArithmeticOperator.srv` (일회성 요청/응답)  
  * **3\. 액션 (Action):** `ArithmeticChecker.action` (장기 실행 목표 및 피드백)

## \[슬라이드 3\] 5.2. 인터페이스 패키지 만들기

### msg, srv, action 파일 생성 및 구조 비교

* **패키지 및 디렉토리 생성:** `msg_srv_action_interface_example` 패키지를 만들고 내부에 `msg/`, `srv/`, `action/` 폴더를 각각 생성합니다.  
* **인터페이스 구조 비교:** 구분선(`---`)의 개수로 요청/응답/피드백 영역을 분리합니다.

| 구분 | 확장자 | 데이터 구조 특징 | 파일 내용 구성 요소 |
| :---- | :---- | :---- | :---- |
| **토픽** | `*.msg` | 단일 데이터 (`---` 없음) | 데이터 필드 |
| **서비스** | `*.srv` | 요청(Request)과 응답(Response) | `Request`  \`---\` \`Response\` |
| **액션** | `*.action` | 목표(Goal), 결과(Result), 피드백(Feedback) | `Goal`  \`---\` \`Result\` \`---\` \`Feedback\` |

## \[슬라이드 4\] 5.3. 패키지 설정 파일 (package.xml)

### 인터페이스 생성에 필요한 필수 의존성 추가

* **역할:** 패키지가 커스텀 인터페이스를 빌드하고 실행하기 위해 필요한 ROS 2 핵심 도구들을 선언합니다.  
* **필수 추가 의존성 패키지:**  
  * `<buildtool_depend>`: `rosidl_default_generators` (인터페이스 코드 생성기)  
  * `<exec_depend>`: `builtin_interfaces` (표준 시간 등의 내장 인터페이스)  
  * `<exec_depend>`: `rosidl_default_runtime` (실행 시 필요한 런타임 라이브러리)  
  * `<member_of_group>`: `rosidl_interface_packages` (인터페이스 그룹 등록)

## \[슬라이드 5\] 5.4. 빌드 설정 파일 (CMakeLists.txt)

### 인터페이스 컴파일 및 언어별 코드 자동 생성 설정

* **역할:** 작성한 `.msg`, `.srv`, `.action` 파일들을 C++ 및 Python 환경에서 사용할 수 있도록 빌드 시스템에 등록합니다.  
* **주요 설정 단계:**  
  1. **의존성 로드:** `find_package(rosidl_default_generators REQUIRED)`  
  2. **파일 지정:** `set(msg_files "msg/ArithmeticArgument.msg")` 등 배열 형태로 파일 경로 저장  
  3. **인터페이스 생성 함수 호출:**  
       
     rosidl\_generate\_interfaces(${PROJECT\_NAME}  
       
       ${msg\_files} ${srv\_files} ${action\_files}  
       
       DEPENDENCIES builtin\_interfaces)

## \[슬라이드 6\] 5.5. 빌드하기

### Colcon 빌드 및 언어별 라이브러리 자동 생성 확인

* **빌드 실행:** 워크스페이스로 이동(`cd ~/robot_ws`) 후 특정 패키지만 선택하여 빌드합니다. (alias `cbp` 활용 시 속도 향상)  
  * `$ cbp msg_srv_action_interface_example`  
* **빌드 결과 (자동 생성 구조):** 인터페이스 정의 파일만 작성해도, 빌드 시 언어별 사용 코드가 지정된 폴더에 **자동으로 생성**됩니다.  
  * `include/`: C++에서 `#include` 하여 사용할 헤더 파일 생성  
  * `lib/python3.x/site-packages/`: Python에서 `import` 하여 사용할 모듈 생성  
  * `share/`: 원본 정의 파일(`msg`, `srv`, `action`) 보관
