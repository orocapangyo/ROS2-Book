## 1.1. 코드 스타일 가이드
- 가독성과 협업 효율이 높아지고, 유지보수와 리뷰가 훨씬 쉬워진다.

## 1.2. 기본 이름 규칙
### 이름규칙
- 파일 이름 및 변수명, 함수명에는 모두 소문자로 snake_case 규칙 사용
- 확장자는 모두 소문자를 사용
- 타입 및 클래스는 CamelCased 사용

### 상수표기
- ALL_CAPITALS 는 UPPERCASE 를 사용하고 모든 글자를 대문자로 사용
```c
int32 MAX_SPEED=10
string MODE_AUTO="auto"
```

### 고유의 이름(예외)
- “이름 규칙의 예외”라기보다 도구가 요구하는 고유명사 라서 별도 예외처럼 보이는 것.
```
package.xml
CMakeLists.txt
README.md
LICENSE
CHANGELOG.rst
.gitignore
.travis.yml
*.repos
```

## 1.3. C++ Style
### 기본규칙
- C++14 Standard를 준수한다.
### 라인 길이
- 최대 100문자
### 이름규칙
- CamelCased : 타입, 클래스, 구조체, 열거형
- snake_case : 파일, 패키지, 인터페이스, 네임스페이스, 변수, 함수, 메소드
- ALL_CAPITALS : 상수, 매크로
- 소스파일 : cpp
- 헤더파일 : hpp
- 전역변수 : (g_)
- 클래스 멤버변수 : (_)
### 공백 문자 대 탭 
- 2 space key
- public, protected, private(class 접근 지정자)는 들여쓰기 금지
### 괄호(Brace)
- if, else, do, while, for 구문에 괄호
- 긴 조건문의 경우의 소괄호/중괄호 사용법
```cpp
if (
    this && that || both && this && that || both && this && that || both && this && that)
{
    (생략)
}
```
- 짧은 함수 호출문
```cpp
call_func(foo, bar);

```
- 긴 함수 호출문에서 소괄호 사용법
```cpp
call_func(
    foo, bar, foo, bar, foo, bar, foo, bar, foo, bar, foo, bar, foo, bar, foo, bar,
    foo, bar, foo, bar, foo, bar, foo, bar, foo, bar, foo, bar, foo, bar, foo, bar);

```
- 매우 긴 함수 인수를 사용해야하는 경우, 가독성을 위해 개행함
```cpp
call_func(
    bang,
    fooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo,
    bar, bat);

```
- 함이름이 길때 2스페이스로 매개변수를 다음 줄로 내린다.
```cpp
ReturnType LongClassName::ReallyReallyReallyLongFunctionName(
    Type par_name1,  // 2 스페이스 들여쓰기
    Type par_name2,
    Type par_name3)
{
      DoSomething();  // 2 스페이스 들여쓰기
  (생략)
}

```
- 첫 구문은 들여쓰기를 사용하지 않고, 이후부터는 2 스페이스 들여쓰기
```cpp
MyClass::MyClass(int var)
: some_var_(var),  // 첫 구문은 들여쓰기를 사용하지 않고, 이후부터는 2 스페이스 들여쓰기
  some_other_var_(var + 1)
{
  (생략)
  DoSomething();
  (생략)
}

```
### 주석
- /** */
-  //

### 린터(Linters)
- 스타일 검사: `ament_cpplint`, `ament_uncrustify`
- 정적 분석: `ament_cppcheck`

### 기타
- Boost 라이브러리 사용은 가능한 피하고 어쩔 수 없을 경우에만 사용.
- 포인터 구문은 char * c처럼 사용한다(char* c나 char *c처럼 사용하지 않는다).
- 중첩 템플릿은 set<list<string>>처럼 사용.


## 1.4. Python Style
- PEPs(Python Enhancement Proposals)의 PEP 8(규칙문서)을 준수

### 기본규칙
- 파이썬 3.5 이상을 사용.
### 라인 길이
- 최대 100문자
### 이름규칙
- CamelCased : 타입, 클래스
- snake_case : 파일, 패키지, 인터페이스, 모듈, 함수, 메소드
- ALL_CAPITALS : 상수
### 공백 문자 대 탭 
- 4 space
- Hanging indent(내어쓰기)
```python
foo = function_name(var_one, var_two, var_three, var_four)

def long_long_long_long_function_name(    
        var_one,
        var_two,
        var_three,
        var_four):
    print(var_one)

```
### 괄호(Brace)
```python
list = [1, 2, 3, 4, 5]
dictionary = {'age': 30, 'name': '홍길동'}
tuple = (1, 2, 3, 4, 5)

```

### 주석
- Docstring Conventions : """ 
- **#** 사용
- 
### 린터(Linters)
- ament_flake8 사용

### 기타
- 모든 문자는 ` 사용
