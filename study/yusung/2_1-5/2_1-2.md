## 2.4. 퍼블리셔 노드 작성
- publisher.py의 import 구문
- Node 클래스 사용, QoSProfile 클래스 사용, std_msgs.msg 모듈의 String 메시지 인터페이스 사용
```python
import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile
from std_msgs.msg import String
```

- 노드의 메인 클래스 
```python
class HelloworldPublisher(Node):
```

- 부모클래스(Node)의 생성자 호출 과 노드 이름 helloworld_publisher 지정
```python
    def __init__(self): # 클래스가 생성될 때 자동으로 실행되는 생성자
        super().__init__('helloworld_publisher') # 부모 클래스(Node) 생성자를 호출하고 노드 이름을 'helloworld_publisher'로 설정
        qos_profile = QoSProfile(depth=10) # 메시지를 최대 10개까지 저장하는 QoS 프로파일 생성
        self.helloworld_publisher = self.create_publisher(String, 'helloworld', qos_profile)  # String 타입 메시지를 'helloworld' 토픽으로 발행하는 퍼블리셔 생성
        self.timer = self.create_timer(1, self.publish_helloworld_msg) # 1초마다 publish_helloworld_msg 함수를 실행하는 타이머 생성
        self.count = 0 # 발행 횟수 또는 메시지 번호를 저장하기 위한 변수 초기화
```

- 콜백 함수의 구현 member function, lambda, local function
- member function 방식 
```python
    def publish_helloworld_msg(self): # Hello World 메시지를 생성해서 토픽으로 발행하는 함수
        msg = String() # String 타입의 ROS2 메시지 객체를 생성
        msg.data = 'Hello World: {0}'.format(self.count) # 메시지 데이터에 count 값을 포함한 문자열을 저장
        self.helloworld_publisher.publish(msg)  # 생성한 메시지를 퍼블리셔를 통해 발행
        self.get_logger().info('Published message: {0}'.format(msg.data)) # 발행한 메시지 내용을 로그로 출력
        self.count += 1 # 다음 메시지를 위해 count 값을 1 증가
```

- main 작성
```python
def main(args=None): # ROS2 노드를 실행하기 위한 메인 함수 정의
    rclpy.init(args=args) # ROS2 Python 클라이언트 라이브러리를 초기화
    node = HelloworldPublisher() # HelloworldPublisher 노드 객체를 생성
    try: # 예외 발생 가능 구간을 처리하기 위한 try 블록 시작
        rclpy.spin(node) # 노드를 계속 실행시키며 콜백 함수가 동작하도록 대기
    except KeyboardInterrupt: # Ctrl+C 입력으로 인한 KeyboardInterrupt 예외를 처리
        node.get_logger().info('Keyboard Interrupt (SIGINT)')  # 종료 신호가 들어왔음을 로그로 출력
    finally: # 예외 발생 여부와 관계없이 항상 실행되는 블록
        node.destroy_node() # 생성한 노드 객체를 명시적으로 삭제
        rclpy.shutdown() # ROS2 통신 시스템을 종료


if __name__ == '__main__': # 현재 파일이 직접 실행된 경우에만 아래 코드를 실행
    main() # main 함수를 호출하여 프로그램 시작
```

## 2.5. 서브스크라이버 노드 작성
- publisher.py의 import 구문
- Node 클래스 사용, QoSProfile 클래스 사용, std_msgs.msg 모듈의 String 메시지 인터페이스 사용
```python
import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile
from std_msgs.msg import String
```
- Node 클래스 상속
```python
class HelloworldSubscriber(Node):
```

- helloworld 라는 토픽을 구독하는 Subscriber를 만드는 부분
- 토픽의 문자열 메시지를 받아서 처리할 준비를 하는 코드
```python
def __init__(self):  # 노드 객체가 만들어질 때 실행되는 초기화 함수이다.
    super().__init__('Helloworld_subscriber')  # 이 노드의 이름을 'Helloworld_subscriber'로 설정한다.
    qos_profile = QoSProfile(depth=10)  # 수신 메시지를 10개까지 버퍼에 보관할 수 있도록 QoS를 설정한다.
    self.helloworld_subscriber = self.create_subscription(  # 'helloworld' 토픽을 구독하는 서브스크라이버를 생성한다.
        String,  # 토픽에서 주고받는 메시지 형식은 String이다.
        'helloworld',  # 구독 대상 토픽 이름이다.
        self.subscribe_topic_message,  # 메시지를 수신하면 이 콜백 함수가 호출된다.
        qos_profile)  # 위에서 만든 QoS 설정을 적용한다.
```

- 토픽에서 메시지를 받으면 그 내용을 화면에 출력하는 함수
```python
def subscribe_topic_message(self, msg):  # 구독한 토픽의 메시지를 받아 처리하는 콜백 함수
    self.get_logger().info('Received message: {0}'.format(msg.data))  # 받은 메시지 내용을 로그에 출력
```

- Subscriber 노드를 실행하고, 종료될 때 안전하게 정리하는 코드
```python
def main(args=None):  # ROS2 노드를 실행하는 메인 함수
    rclpy.init(args=args)  # ROS2 Python 기능을 초기화
    node = HelloworldSubscriber()  # Subscriber 노드 객체를 생성
    try:  # 예외가 발생할 수 있는 실행 구간 시작
        rclpy.spin(node)  # 노드를 계속 실행하며 메시지를 기다림
    except KeyboardInterrupt:  # Ctrl+C로 종료할 때 예외 처리
        node.get_logger().info('Keyboard Interrupt (SIGINT)')  # 종료 신호가 들어왔음을 로그로 출력
    finally:  # 프로그램 종료 시 항상 실행되는 정리 구간
        node.destroy_node()  # 노드 객체를 삭제
        rclpy.shutdown()  # ROS2 시스템 종료
if __name__ == '__main__':  # 현재 파일이 직접 실행될 때만 아래 코드 실행
    main()  # main 함수 호출
```

## 2.6. 빌드
- 워크스페이스 내의 모든 패키지 빌드하는 방법
```bash
$ cd ~/robot_ws && colcon build --symlink-install
```

- 특정 패키지만 빌드하는 방법
```bash
$ cd ~/robot_ws && colcon build --symlink-install --packages-select [패키지 이름1] [패키지 이름2] [패키지 이름N]
```

- 특정 패키지 및 의존성 패키지를 함께 빌드하는 방법
```bash
$ cd ~/robot_ws && colcon build --symlink-install --packages-up-to [패키지 이름]
```
