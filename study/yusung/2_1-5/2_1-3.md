## 3.3. 패키지 설정
### 3.3.1 패키지 설정 파일
- package.xml
-  c++ 이면 build_type으로 ament_cmake을 설정
```xml
<?xml version="1.0"?> <!-- XML 문서의 버전을 지정 -->
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?> <!-- package.xml이 따르는 ROS2 패키지 스키마 정보를 지정 -->
<package format="3"> <!-- ROS2 패키지 메타정보의 시작이며, format 3 형식을 사용 -->

  <name>my_first_ros_rclcpp_pkg</name> <!-- 패키지 이름 -->
  <version>0.6.0</version> <!-- 패키지 버전 -->
  <description>ROS 2 rclcpp basic package for the ROS 2 seminar</description> <!-- 패키지에 대한 설명 -->
  <maintainer email="passionvirus@gmail.com">Pyo</maintainer> <!-- 패키지 유지보수자 정보 -->
  <license>Apache License 2.0</license> <!-- 패키지의 라이선스 정보 -->

  <author>Mikael Arguedas</author> <!-- 패키지 작성자 -->
  <author>Morgan Quigley</author> <!-- 패키지 작성자 -->
  <author email="jacob@openrobotics.org">Jacob Perron</author> <!-- 이메일이 포함된 패키지 작성자 -->
  <author email="passionvirus@gmail.com">Pyo</author> <!-- 이메일이 포함된 패키지 작성자 -->

  <buildtool_depend>ament_cmake</buildtool_depend> <!-- 빌드 도구로 ament_cmake를 사용함 -->
  <depend>rclcpp</depend> <!-- 실행 및 빌드 시 필요한 의존 패키지 -->
  <depend>std_msgs</depend> <!-- 표준 메시지 패키지 의존성 -->

  <test_depend>ament_lint_auto</test_depend> <!-- 테스트/코드 검사에 사용하는 의존 패키지 -->
  <test_depend>ament_lint_common</test_depend> <!-- 공통 lint 테스트 도구 의존성 -->

  <export> <!-- 빌드 시스템이나 외부 도구에 전달할 추가 정보의 시작 -->
    <build_type>ament_cmake</build_type> <!-- 이 패키지의 빌드 타입이 ament_cmake임을 명시 -->
  </export> <!-- 추가 정보 영역 종료 -->

</package> <!-- ROS2 패키지 메타정보 종료 -->
```

### 3.3.1 빌드 설정 파일
- CMakeLists.txt
- 의존석 패키지의 설정과 빌드 및 설치 관련 설정
- ament_cmake 패키지, rclcpp 패키지, 메시지 인터페이스, std_msg 패키지
- helloworld_publisher 와 helloworld_subscriber 실행파일 빌드 및 설치를 위한 설정이 포함

```text
# Set minimum required version of cmake, project name and compile options
cmake_minimum_required(VERSION 3.5)
project(my_first_ros_rclcpp_pkg)

if(NOT CMAKE_CXX_STANDARD)
  set(CMAKE_CXX_STANDARD 14)
endif()

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# Find dependencies
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(std_msgs REQUIRED)

# Build
add_executable(helloworld_publisher src/helloworld_publisher.cpp)
ament_target_dependencies(helloworld_publisher rclcpp std_msgs)

add_executable(helloworld_subscriber src/helloworld_subscriber.cpp)
ament_target_dependencies(helloworld_subscriber rclcpp std_msgs)

# Install
install(TARGETS
  helloworld_publisher
  helloworld_subscriber
  DESTINATION lib/${PROJECT_NAME})

# Test
if(BUILD_TESTING)
  find_package(ament_lint_auto REQUIRED)
  ament_lint_auto_find_test_dependencies()
endif()

# Macro for ament package
ament_package()
```

## 3.4. 퍼블리셔 노드 작성
- `<chrono>`: 타이머 주기 설정을 위한 시간 관련 기능
- `<functional>`: `std::bind` 등 콜백 연결에 사용
- `<memory>`: 스마트 포인터(`std::shared_ptr`) 사용
- `<string>`: 문자열 처리
- `"rclcpp/rclcpp.hpp"`: ROS 2 C++ API 사용
- `"std_msgs/msg/string.hpp"`: 문자열 메시지 타입 사용
- `using namespace std::chrono_literals;` : `500ms`, `1s` 같은 시간 리터럴을 간단히 쓰기 위해 사용
```cpp
#include <chrono>      // std::chrono 타이머 주기 설정을 위한 시간 관련 기능(예: 1s, 500ms)
#include <functional>  // std::bind 등 콜백 함수 연결에 필요한 기능 사용
#include <memory>      // std::shared_ptr 같은 스마트 포인터 사용
#include <string>      // std::string 문자열 타입 사용

#include "rclcpp/rclcpp.hpp"        // ROS 2 C++ 노드, 퍼블리셔, 타이머 등 핵심 기능 사용
#include "std_msgs/msg/string.hpp"  // std_msgs::msg::String 메시지 타입 사용

using namespace std::chrono_literals;  // 1s, 500ms 같은 시간 리터럴을 간단히 사용
```

- 클래스를 하나 정의하는데, 그 클래스가 ROS 2의 노드 기능을 상속받는다 라는 뜻
```cpp
class HelloworldPublisher : public rclcpp::Node
```

- 한 번 호출되어 초기화 되는 생성자(constructor)
```cpp
public:
  HelloworldPublisher()
  // ROS 2 노드 이름을 "helloworld_publisher"로 설정하고,
  // 메시지 카운터를 0으로 초기화
  : Node("helloworld_publisher"), count_(0)
  {
    // 최근 10개의 메시지를 유지하는 QoS 설정
    auto qos_profile = rclcpp::QoS(rclcpp::KeepLast(10));

    // "helloworld" 토픽으로 String 메시지를 발행하는 퍼블리셔 생성
    helloworld_publisher_ = this->create_publisher<std_msgs::msg::String>(
      "helloworld", qos_profile);

    // 1초마다 publish_helloworld_msg()를 호출하는 타이머 생성
    timer_ = this->create_wall_timer(
      1s, std::bind(&HelloworldPublisher::publish_helloworld_msg, this));
  }
```

```cpp
private:
  // 타이머에 의해 주기적으로 호출되는 함수 (메시지 발행 역할)
  void publish_helloworld_msg()
  {
    auto msg = std_msgs::msg::String();  // String 타입 메시지 객체 생성

    // "Hello World: N" 형태로 메시지 구성 (count_는 발행할 때마다 증가)
    msg.data = "Hello World: " + std::to_string(count_++);

    // 콘솔에 현재 발행한 메시지 출력 (디버깅/확인용)
    RCLCPP_INFO(this->get_logger(), "Published message: '%s'", msg.data.c_str());

    // 실제로 토픽에 메시지 발행
    helloworld_publisher_->publish(msg);
  }

  // 타이머 객체 (주기적으로 콜백 함수 실행을 담당)
  rclcpp::TimerBase::SharedPtr timer_;

  // 퍼블리셔 객체 (helloworld 토픽으로 메시지 발행)
  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr helloworld_publisher_;

  // 메시지 번호 카운터 (발행 횟수 추적용)
  size_t count_;
```

- 콜백 내용이 짧을 때, 다른곳에서 재사용하지 않을 때 lambda 구현
```cpp
timer_ = this->create_wall_timer(
  1s,  // 1초마다 아래 lambda 콜백 함수 실행
  [this]() -> void
  {
    auto msg = std_msgs::msg::String();  // String 타입 메시지 객체 생성

    // "Hello World2: N" 형태의 메시지 데이터 설정
    msg.data = "Hello World2: " + std::to_string(count_++);

    // 현재 발행한 메시지를 콘솔에 출력
    RCLCPP_INFO(this->get_logger(), "Published message: '%s'", msg.data.c_str());

    // helloworld 토픽으로 메시지 발행
    helloworld_publisher_->publish(msg);
  }
);
```

```cpp
int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);  // ROS 2 통신 시스템 초기화
  auto node = std::make_shared<HelloworldPublisher>();  // HelloworldPublisher 노드 객체 생성
  rclcpp::spin(node);  // 노드를 실행 상태로 유지하며 콜백 함수 처리
  rclcpp::shutdown();  // ROS 2 종료 및 자원 정리
  return 0;  // 프로그램 정상 종료
}
```

## 3.5. 서브스크라이버 노드 작성

```cpp
#include <functional>  // std::bind, std::placeholders::_1 등 콜백 함수 연결에 필요한 기능 사용
#include <memory>      // std::shared_ptr 같은 스마트 포인터 사용

#include "rclcpp/rclcpp.hpp"        // ROS 2 C++ 노드, 구독자, 퍼블리셔 등 핵심 기능 사용
#include "std_msgs/msg/string.hpp"  // std_msgs::msg::String 메시지 타입 사용

using std::placeholders::_1;  // std::bind에서 콜백 함수의 첫 번째 인자를 의미하는 자리표시자
```

```cpp
class HelloworldSubscriber : public rclcpp::Node  // ROS 2 Node를 상속받는 HelloworldSubscriber 클래스 정의
```

```cpp
public:
  HelloworldSubscriber()
  : Node("Helloworld_subscriber")  // ROS 2 노드 이름을 "Helloworld_subscriber"로 설정
  {
    auto qos_profile = rclcpp::QoS(rclcpp::KeepLast(10));  // 최근 10개의 메시지를 유지하는 QoS 설정

    helloworld_subscriber_ = this->create_subscription<std_msgs::msg::String>(
      "helloworld",  // 구독할 토픽 이름
      qos_profile,   // 토픽 통신에 사용할 QoS 설정
      std::bind(&HelloworldSubscriber::subscribe_topic_message, this, _1));  // 메시지가 오면 콜백 함수 실행
  }
```

```cpp
private:
  // 토픽 메시지를 수신했을 때 자동으로 호출되는 콜백 함수
  void subscribe_topic_message(const std_msgs::msg::String::SharedPtr msg) const
  {
    // 수신한 메시지 내용을 콘솔에 출력
    RCLCPP_INFO(this->get_logger(), "Received message: '%s'", msg->data.c_str());
  }

  // "helloworld" 토픽을 구독하는 subscriber 객체
  rclcpp::Subscription<std_msgs::msg::String>::SharedPtr helloworld_subscriber_;
```

```cpp
int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);  // ROS 2 통신 시스템 초기화
  auto node = std::make_shared<HelloworldSubscriber>();  // HelloworldSubscriber 노드 객체 생성
  rclcpp::spin(node);  // 노드를 계속 실행하며 들어오는 메시지 콜백 처리
  rclcpp::shutdown();  // ROS 2 종료 및 자원 정리
  return 0;  // 프로그램 정상 종료
}
```

## 3.6. 빌드
- colcon 빌드 툴 사용

```bash
(워크스페이스 내의 모든 패키지를 빌드하는 방법)
$ cd ~/robot_ws && colcon build --symlink-install

(특정 패키지만 빌드하는 방법)
$ cd ~/robot_ws && colcon build --symlink-install --packages-select [패키지 이름1] [패키지 이름2] [패키지 이름N]

(특정 패키지 및 의존성 패키지를 함께 빌드하는 방법)
$ cd ~/robot_ws && colcon build --symlink-install --packages-up-to [패키지 이름]
```

- ~/robot_ws 작업공간으로 이동
- my_first_ros_rclcpp_pkg 패키지만 선택해서 빌드
```bash
$ cd ~/robot_ws
$ colcon build --symlink-install --packages-select my_first_ros_rclcpp_pkg
Starting >>> my_first_ros_rclcpp_pkg
Finished <<< my_first_ros_rclcpp_pkg [2.53s]
Summary: 1 package finished [2.76s]
```

- 현재 터미널에 ~/robot_ws 워크스페이스 환경설정을 적용
- 이 작업을 해야 방금 빌드한 패키지를 ROS 2가 인식할 수 있음
```bash
. ~/robot_ws/install/local_setup.bash
```
