## 4.1. 설정 스크립트(setup script)
- ~/.bashrc에 저장 후 사용
```bash
$ source ~/robot_ws/install/local_setup.bash
```

## 4.2. setup.bash vs local_setup.bash
- 각각 사용하는 목적이 다름
- setup.bash = 전체 환경 초기화
- local_setup.bash = 현재 레이어만 추가
  
### 4.2.1 underlay 와 overlay
- 바이너리 설치 후 /opt/ros/jazzy 설치위치

### 4.2.2 setup.bash 및 local_setup.bash 사용 방법
- ./bashrc 정보에 넣어두고 사용하면 편하다
```bash
$ source /opt/ros/jazzy/setup.bash
$ source ~/robot_install/local_setup.bash
```

## 4.3. colcon_cd
- ./bashrc 추가해서 사용
```bash
$ colcon_cd 패키지이름
$ source /usr/share/colcon_cd/function/colcon_cd.sh
$ export _colcon_cd_root=~/robot_ws
```

## 4.4. ROS_DOMAIN_ID vs Namespace
- 동일 네트워크를 다른사람과 공유
- 1) 물리적으로 다른 네트워크를 사용
- 2) ROS_DOMAIN_ID를 이용하여 DDS의 domain을 변경
- 3) 각 노드 및 토픽/서비스/액션의 이름에 Namespace 추가

### 4.4.1 물리적으로 다른 네트워크
- 독립적인 스위치 허브 및 라우터 사용 추천

### 4.4.2 ROS_DOMAIN_ID
- 환경변수 설정, 0 ~ 101까지 사용
```bash
$ export ROS_DOMAIN_ID=11
$ ros2 run demo_nodes_cpp talker
[INFO]: Publishing: 'Hello World: 1'
[INFO]: Publishing: 'Hello World: 2'
[INFO]: Publishing: 'Hello World: 3'
(생략)
```

- 반응 없음
```bash
$ export ROS_DOMAIN_ID=12
$ ros2 run demo_nodes_cpp listener
```

- 수신
```bash
$ export ROS_DOMAIN_ID=11
$ ros2 run demo_nodes_cpp listener
[INFO]: I heard: [Hello World: 13]
[INFO]: I heard: [Hello World: 14]
[INFO]: I heard: [Hello World: 15]
(생략)
```

### 4.4.3 ROS 2 Namespace
- ns(namespace)
```bash
$ ros2 run turtlesim turtlesim_node
```

- ns 확인, 토픽 확인
```bash
$ rqt_graph
``` 

- ns 확인, 토픽 확인
``` bash 
$ ros2 run turtlesim turtlesim_node --ros-args -r __ns:=/robot1
$ ros2 run turtlesim turtlesim_node --ros-args -r __ns:=/robot2
$ rqt_graph
```
