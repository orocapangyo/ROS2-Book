

### 5.2.1 ArithmeticArgument.msg 생성

### 5.2.2 ArithmeticOperator.srv 생성
- msg_srv_action_interface_example/srv/ArithmeticOperator.srv
```bash
# Constants
int8 PLUS = 1
int8 MINUS = 2
int8 MULTIPLY = 3
int8 DIVISION = 4

# Request
int8 arithmetic_operator
---
# Response
float32 arithmetic_result
```

### 5.2.3 ArithmeticChecker.action 생성
- msg_srv_action_interface_example/action/ArithmeticChecker.action
```bash
# action/ArithmeticChecker.action

# Goal
float32 goal_sum
---
# Result
string[] all_formula
float32 total_sum
---
# Feedback
string[] formula
```
### 5.2.4 msg, srv, action 인터페이스 비교
- 데이터, 형식, 사용 예로 구분

| 구분 | msg 인터페이스 | srv 인터페이스 | action 인터페이스 |
|---|---|---|---|
| 확장자 | `*.msg` | `*.srv` | `*.action` |
| 데이터 | 토픽 데이터(data) | 서비스 요청(request)<br>`---`<br>서비스 응답(response) | 액션 목표(goal)<br>`---`<br>액션 결과(result)<br>`---`<br>액션 피드백(feedback) | 
| 구분 | msg 인터페이스 | srv 인터페이스 | action 인터페이스 |
| 형식 | `fieldtype1 fieldname1`<br>`fieldtype2 fieldname2`<br>`fieldtype3 fieldname3` | `fieldtype1 fieldname1`<br>`fieldtype2 fieldname2`<br>`---`<br>`fieldtype3 fieldname3`<br>`fieldtype4 fieldname4` | `fieldtype1 fieldname1`<br>`fieldtype2 fieldname2`<br>`---`<br>`fieldtype3 fieldname3`<br>`fieldtype4 fieldname4`<br>`---`<br>`fieldtype5 fieldname5`<br>`fieldtype6 fieldname6` |
| 구분 | msg 인터페이스 | srv 인터페이스 | action 인터페이스 |
| 사용 예 | `[ArithmeticArgument.msg]`<br><br>`builtin_interfaces/Time stamp`<br>`float32 argument_a`<br>`float32 argument_b` | `[ArithmeticOperator.srv]`<br><br>`# Constants`<br>`int8 PLUS = 1`<br>`int8 MINUS = 2`<br>`int8 MULTIPLY = 3`<br>`int8 DIVISION = 4`<br><br>`# Request`<br>`int8 arithmetic_operator`<br>`---`<br>`# Response`<br>`float32 arithmetic_result` | `[ArithmeticChecker.action]`<br><br>`# Goal`<br>`float32 goal_sum`<br>`---`<br>`# Result`<br>`string[] all_formula`<br>`float32 total_sum`<br>`---`<br>`# Feedback`<br>`string[] formula` |

## 5.3. 패키지 설정 파일(package.xml)

```bash
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>msg_srv_action_interface_example</name>
  <version>0.6.0</version>
  <description>
    ROS 2 example for message, service and action interface
  </description>
  <maintainer email="passionvirus@gmail.com">Pyo</maintainer>
  <license>Apache 2.0</license>
  <author email="passionvirus@gmail.com">Pyo</author>
  <author email="routiful@gmail.com">Darby Lim</author>
  <buildtool_depend>ament_cmake</buildtool_depend>
  <buildtool_depend>rosidl_default_generators</buildtool_depend>
  <exec_depend>builtin_interfaces</exec_depend>
  <exec_depend>rosidl_default_runtime</exec_depend>
  <member_of_group>rosidl_interface_packages</member_of_group>
  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

- 필수적인 의존성 패키지
```bash
<buildtool_depend>rosidl_default_generators</buildtool_depend>
  <exec_depend>builtin_interfaces</exec_depend>
  <exec_depend>rosidl_default_runtime</exec_depend>
``` 


## 5.4. 빌드 설정 파일(CMakeLists.txt)
```bash
################################################################################
# Set minimum required version of cmake, project name and compile options
################################################################################
cmake_minimum_required(VERSION 3.5)
project(msg_srv_action_interface_example)

if(NOT CMAKE_CXX_STANDARD)
  set(CMAKE_CXX_STANDARD 14)
endif()

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -Wall -Wextra -Wpedantic")
endif()

################################################################################
# Find and load build settings from external packages
################################################################################
find_package(ament_cmake REQUIRED)
find_package(builtin_interfaces REQUIRED)
find_package(rosidl_default_generators REQUIRED)

################################################################################
# Declare ROS messages, services and actions
################################################################################
set(msg_files
  "msg/ArithmeticArgument.msg"
)

set(srv_files
  "srv/ArithmeticOperator.srv"
)

set(action_files
  "action/ArithmeticChecker.action"
)

rosidl_generate_interfaces(${PROJECT_NAME}
  ${msg_files}
  ${srv_files}
  ${action_files}
  DEPENDENCIES builtin_interfaces
)

################################################################################
# Macro for ament package
################################################################################
ament_export_dependencies(rosidl_default_runtime)
ament_package()
```
- rosidl_generate_interfaces에 해당 set들을 기입

## 5.5. 빌드하기
- cw : `cd ~/robot_ws`의 줄임말 (워크스페이스 빠르게 이동)
- cbp : 특정 패키지만 빌드할 때 사용 `colcon build --symlink-install --packages-select <패키지명>`
- alias 명령

```bash
$ CW
$ cbp msg_srv_action_interface_example
```

- 빌드 후 생성된 인터페이스 패키지 구조, 모두 자동 생성됨
- `include/` : C++에서 사용하는 헤더 파일 생성 위치
- `lib/python3.x/site-packages/` : Python에서 import 가능한 모듈 생성 위치
- `share/` : 인터페이스 정의 파일(msg/srv/action) 원본 저장 위치
- 인터페이스만 정의하면 언어별 코드가 자동으로 만들어지는 구조

```bash
~/robot_ws/install/msg_srv_action_interface_example

├── include
│   └── msg_srv_action_interface_example
│       ├── action
│       ├── msg
│       └── srv
│
├── lib
│   └── python3.6
│       └── site-packages
│           └── msg_srv_action_interface_example
│               ├── action
│               ├── msg
│               └── srv
│
└── share
    └── msg_srv_action_interface_example
        ├── action
        ├── msg
        └── srv
```

- 사용할 인터페이스 파일을 신규로 작성하여 지정된 폴더에 넣어주고 패키지 설정파일 과 빌드 설정파일을 작성
